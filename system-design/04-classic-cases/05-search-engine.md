# 设计搜索引擎

## 需求分析

### 功能需求

1. **全文检索**：支持关键词搜索
2. **拼写纠错**：自动纠正拼写错误
3. **搜索建议**：输入时自动补全
4. **相关性排序**：按相关性返回结果
5. **分页**：支持结果分页
6. **过滤**：按时间、类型等过滤

### 非功能需求

| 需求 | 目标 |
|------|------|
| 可用性 | 99.99% |
| 延迟 | 搜索响应 < 200ms |
| 数据规模 | 支持百亿级文档 |
| QPS | 10万+ |

## 核心概念

### 倒排索引 (Inverted Index)

```
正向索引：文档 → 词
文档1: "Java 编程"
文档2: "Python 编程"

倒排索引：词 → 文档列表
Java → [文档1]
编程 → [文档1, 文档2]
Python → [文档2]
```

**倒排索引结构**：
```
Term      | Document Frequency | Postings List
----------|-------------------|---------------------------------
Java      | 100               | [(doc1, 5, [pos1, pos2]), (doc3, 3, [pos1])]
编程      | 5000              | [(doc1, 2, [pos3]), (doc2, 4, [pos1, pos5])]

Postings List 包含：
- 文档 ID
- 词频 (TF)
- 位置列表
```

### 分词 (Tokenization)

```
英文：
"Hello World" → ["hello", "world"]

中文（复杂）：
"中华人民共和国" → ["中华", "人民", "共和国"] 或 ["中华人民共和国"]

分词算法：
- 基于词典：最大匹配法
- 基于统计：HMM、CRF
- 基于深度学习：BERT
```

### TF-IDF 算法

```
TF (Term Frequency): 词在文档中出现的频率
TF = 词在文档中出现次数 / 文档总词数

IDF (Inverse Document Frequency): 逆文档频率
IDF = log(总文档数 / 包含该词的文档数)

TF-IDF = TF × IDF

意义：
- TF 高：词在文档中重要
- IDF 高：词在所有文档中稀缺
- TF-IDF 高：词对该文档具有区分度
```

## 核心组件

### 1. 爬虫 (Crawler)

```
职责：抓取网页内容

架构：
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  URL Queue  │────►│  Crawler    │────►│  Parser     │
│  (待爬URL)   │     │  (下载网页)  │     │  (解析内容)  │
└─────────────┘     └─────────────┘     └──────┬──────┘
      ▲                                        │
      └────────────────────────────────────────┘
                   (提取新URL)

去重策略：
- Bloom Filter：快速判断URL是否已爬
- URL 指纹：MD5/SHA1
```

### 2. 索引构建 (Indexer)

```
流程：
1. 接收文档
2. 分词
3. 计算 TF-IDF
4. 构建倒排索引
5. 写入索引文件

MapReduce 构建：
Map: (docId, content) → [(word, docId, tf), ...]
Reduce: (word, [(docId, tf), ...]) → 倒排列表
```

### 3. 查询处理 (Query Processor)

```
流程：
1. 分词："Java 编程" → ["java", "编程"]
2. 同义词扩展："编程" → ["编程", "开发", "coding"]
3. 停用词过滤：去掉 "的", "是" 等
4. 拼写纠错："javva" → "java"
5. 意图识别：判断是导航、信息还是交易类搜索
```

### 4. 检索服务 (Search Service)

```
流程：
1. 获取查询词的倒排列表
2. 合并倒排列表（AND/OR/NOT）
3. 计算相关性得分
4. 排序返回 Top N

布尔查询：
"Java AND 编程" → 取两个倒排列表的交集
"Java OR Python" → 取两个倒排列表的并集
```

## 相关性排序

### 传统方法

```
TF-IDF：
score = Σ(tf-idf(word, doc) × query_weight)

BM25（改进版）：
- 考虑文档长度归一化
- 非线性 TF 饱和
- 当前主流算法
```

### 机器学习方法

```
Learning to Rank：

Pointwise：单独为每个文档打分
Pairwise：学习文档对的相对顺序
Listwise：优化整个列表的质量

特征：
- 文档特征：PageRank、新鲜度、点击率
- 查询特征：词频、查询长度
- 交互特征：历史点击率、停留时间
```

## 优化策略

### 1. 索引分片

```
按文档ID分片：
Shard 1: docId 1-1000万
Shard 2: docId 1000万-2000万
...

查询时并行查询所有分片，合并结果
```

### 2. 缓存优化

```
多级缓存：
- L1: 热点查询结果（Redis）
- L2: 倒排列表缓存（本地内存）
- L3: 文档内容缓存（CDN）

缓存策略：
- 缓存 Top 1000 热门查询
- 缓存常见过滤条件组合
```

### 3. 查询优化

```
预处理：
- 拼写纠错：编辑距离、最小编辑距离
- 查询扩展：同义词、相关搜索
- 查询重写：识别实体、意图

结果优化：
- 分页优化：深分页问题（限制最大页数）
- 相关性反馈：根据用户点击调整排序
```

## 面试要点

### Q1: 什么是倒排索引？为什么搜索引擎要用它？

**答**：
倒排索引是词到文档的映射，与传统的文档到词的正向索引相反。使用倒排索引可以快速定位包含特定词的文档，将搜索复杂度从 O(N) 降到 O(1) 或 O(logN)，支持大规模数据的高效检索。

### Q2: 如何处理中文分词？

**答**：
中文分词比英文复杂，因为没有空格分隔。常用方法包括：
1. 基于词典：最大正向/逆向匹配，速度快但新词识别差
2. 基于统计：HMM、CRF，可以识别新词
3. 基于深度学习：BERT 等模型，效果最好但性能较低
实际系统中常采用混合策略，词典匹配为主，统计方法补充。

### Q3: 如何评估搜索结果质量？

**答**：
1. 离线指标：NDCG、MAP、MRR 等衡量排序质量
2. 在线指标：点击率、停留时间、跳出率
3. A/B 测试：对比不同算法的效果
4. 人工评估：标注数据，计算相关性得分
