# 设计推荐系统

## 需求分析

### 功能需求

1. **个性化推荐**：根据用户兴趣推荐内容
2. **多样性**：推荐结果不重复、不单调
3. **实时性**：快速反映用户最新行为
4. **可解释性**：能解释为什么推荐

### 非功能需求

| 需求 | 目标 |
|------|------|
| 延迟 | 推荐接口 < 100ms |
| 吞吐 | 10万 QPS |
| 覆盖率 | 长尾内容能被推荐 |
| 新鲜度 | 新内容能被快速推荐 |

## 召回层 (Recall)

### 目的

从海量物品中快速筛选出几千个候选集，缩小后续计算范围。

### 召回策略

| 策略 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 协同过滤 | 相似用户/物品 | 发现用户潜在兴趣 | 冷启动问题 |
| 基于内容 | 标签匹配 | 解释性强 | 多样性差 |
| 热门推荐 | 全局热门 | 新用户友好 | 马太效应 |
| 向量召回 | Embedding 相似度 | 语义匹配 | 计算量大 |

### 协同过滤 (Collaborative Filtering)

```
用户协同过滤 (User-CF)：
- 找到与目标用户相似的用户
- 推荐这些用户喜欢的物品

物品协同过滤 (Item-CF)：
- 找到与目标用户历史喜欢的物品相似的物品
- 推荐这些相似物品

相似度计算：
- 余弦相似度
- Pearson 相关系数
- Jaccard 相似度
```

### 矩阵分解 (Matrix Factorization)

```
用户-物品评分矩阵 R (m × n)：
        物品1  物品2  物品3  ...
用户1    5     3      ?
用户2    4     ?      2
用户3    ?     5      4

分解为：
R ≈ P × Q^T

P: 用户矩阵 (m × k) - 用户隐向量
Q: 物品矩阵 (n × k) - 物品隐向量

预测评分：r_ui = p_u · q_i
```

## 排序层 (Ranking)

### 粗排

```
目的：快速从几千候选集中筛选出几百个

方法：
- 轻量级模型（LR、GBDT）
- 简单特征
- 快速打分
```

### 精排

```
目的：精确排序，选出 Top N

深度学习方法：
1. Wide & Deep
   - Wide: 记忆能力（历史特征）
   - Deep: 泛化能力（Embedding）

2. DeepFM
   - 结合 FM 和 DNN
   - 自动特征交叉

3. DIN (Deep Interest Network)
   - 引入注意力机制
   - 对历史行为加权

特征工程：
- 用户特征：年龄、性别、活跃度
- 物品特征：类别、标签、发布时间
- 交叉特征：用户对类别的偏好
- 上下文特征：时间、地理位置
```

## 重排层 (Rerank)

### 目的

调整精排结果，满足业务需求。

### 策略

```
1. 多样性 (Diversity)：
   - MMR (Maximal Marginal Relevance)
   - 避免推荐内容过于相似

2. 新鲜度 (Freshness)：
   - 提升新内容的权重
   - 探索与利用 (Exploration & Exploitation)

3. 疲劳度 (Fatigue)：
   - 降低用户多次不点击的内容权重
   - 轮播展示

4. 业务规则：
   - 置顶
   - 过滤
   - 加权
```

## 特征工程

### 用户画像

```
人口属性：
- 年龄、性别、地域、职业

兴趣标签：
- 从行为数据提取（点击、收藏、购买）
- 标签权重 = 行为强度 × 时间衰减

行为序列：
- 最近 N 次行为记录
- 用于序列建模（RNN、Transformer）
```

### 物品画像

```
基础属性：
- 类别、标签、作者、发布时间

统计特征：
- 点击率、收藏率、分享率
- 热度分（时间衰减）

Embedding：
- 文本 Embedding（BERT）
- 图像 Embedding（ResNet）
- 协同过滤 Embedding
```

## 评估指标

## A/B 测试

```
流程：
1. 划分流量：随机分成多组
2. 对照组：使用旧算法
3. 实验组：使用新算法
4. 观察指标：CTR、停留时长等
5. 统计显著性：p-value < 0.05
6. 决策：全量或下线

注意事项：
- 样本量足够
- 实验时间足够（覆盖不同时间段）
- 避免辛普森悖论
```

## 面试要点

### Q1: 推荐系统如何解决冷启动问题？

**答**：
1. 新用户：
   - 引导选择兴趣标签
   - 推荐热门内容
   - 基于人口属性的推荐
2. 新物品：
   - 内容-based 推荐
   - 探索机制（EE 问题）
   - 给新物品流量扶持

### Q2: 如何平衡推荐的准确性和多样性？

**答**：
1. 多路召回：协同过滤 + 内容 + 热门，保证多样性来源
2. 重排层引入多样性算法（MMR）
3. 控制同一类别/作者的出现次数
4. A/B 测试找到最佳平衡点

### Q3: 如何实现推荐系统的实时性？

**答**：
1. 实时特征：使用 Flink 计算用户实时行为特征
2. 实时召回：基于实时行为的相似物品召回
3. 在线学习：模型实时更新（FTRL）
4. 缓存策略：热点数据缓存，实时数据查询
