# 常用计算

## 时间单位

```
1 秒 = 1000 毫秒 (ms)
1 毫秒 = 1000 微秒 (μs)
1 微秒 = 1000 纳秒 (ns)

常用延迟参考：
- L1 缓存：1 ns
- 内存访问：100 ns
- SSD 随机读：10 μs
- 磁盘寻道：10 ms
- 网络(同城)：1 ms
- 网络(跨洲)：150 ms
```

## 数据单位

```
1 Byte = 8 bits
1 KB = 1024 Bytes
1 MB = 1024 KB
1 GB = 1024 MB
1 TB = 1024 GB
1 PB = 1024 TB

字符占用：
- ASCII：1 Byte
- UTF-8 中文：3 Bytes
- UTF-8 Emoji：4 Bytes

数据大小估算：
- 用户ID (Long)：8 Bytes
- 时间戳：8 Bytes
- 短文本 (140 字)：~500 Bytes
- 长文本 (1KB)：~1 KB
- 图片 (压缩后)：100 KB - 1 MB
- 视频 (1分钟)：10-100 MB
```

## QPS 计算

```
公式：QPS = 日活 × 人均日操作次数 / 86400

示例：
- DAU 1000万，每人每天发 1 条微博
- QPS = 10,000,000 × 1 / 86400 ≈ 116
- 峰值 QPS ≈ 116 × 3 = 350

常见场景：
- 读 QPS 通常是写 QPS 的 10-100 倍
- 峰值通常是平均值的 2-5 倍
```

## 存储计算

```
公式：总存储 = 日新增数据量 × 保留天数

示例：
- 日新增：1000万条，每条 500 Bytes
- 日存储：10,000,000 × 500B = 5 GB
- 保留 3 年：5 GB × 365 × 3 = 5.5 TB
- 考虑副本 (3副本)：5.5 TB × 3 = 16.5 TB
```

## 带宽计算

```
入站带宽 = 写 QPS × 平均请求大小
出站带宽 = 读 QPS × 平均响应大小

示例：
- 读 QPS：10,000
- 平均响应：10 KB
- 出站带宽 = 10,000 × 10 KB = 100 MB/s = 800 Mbps
```

## 服务器估算

```
Web 服务器：
- 单机 QPS：1000-5000（取决于业务复杂度）
- 假设 2000 QPS，需要 10万/2000 = 50 台

数据库服务器：
- MySQL 单机 QPS：3000-8000
- 需要主从复制、读写分离

Redis 服务器：
- 单机 QPS：5万-10万
- 主要受网络带宽限制
```

## 可用性计算

```
可用性 = (1 - 宕机时间 / 总时间) × 100%

可用性等级：
- 99% (2个9)：年宕机 3.65 天
- 99.9% (3个9)：年宕机 8.76 小时
- 99.99% (4个9)：年宕机 52.6 分钟
- 99.999% (5个9)：年宕机 5.26 分钟

多可用区提升：
- 单可用区 99.9%
- 双可用区 (并行)：99.9999%
- 双可用区 (串行)：99.8%
```

## 哈希空间

```
Base62 (0-9a-zA-Z)：
- 6 位：62^6 = 568 亿
- 7 位：62^7 = 3.5 万亿
- 8 位：62^8 = 218 万亿

MD5：128 位，32 位十六进制表示
SHA-256：256 位，64 位十六进制表示

一致性哈希环：2^32 个槽位
```
