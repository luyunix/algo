# TX001 - 分发饼干 题解

## 思路是怎么想出来的

### 第一步：明确问题

```
问题：用饼干满足孩子，求最多能满足几个孩子

约束：
- 每个孩子最多一块饼干
- 饼干 j 满足孩子 i 的条件：s[j] >= g[i]
- 目标：最大化满足的孩子数

示例：g = [1,2,3], s = [1,1]
- 孩子胃口：1, 2, 3
- 饼干尺寸：1, 1
- 只有尺寸1的饼干，只能满足胃口1的孩子
- 答案：1
```

### 第二步：发现贪心策略

```
直觉：
- 大饼干应该给胃口大的孩子（否则浪费）
- 小饼干给胃口小的孩子

贪心策略：
1. 把孩子和饼干都排序
2. 用最小的饼干满足最小的胃口
3. 如果当前饼干不能满足当前孩子，换更大的饼干
4. 如果能满足，计数+1，继续下一个孩子和饼干

为什么正确？
- 小饼干给小胃口：不浪费大饼干
- 如果小饼干连小胃口都满足不了，肯定满足不了大胃口
- 证明：交换论证，如果最优解没这样匹配，可以调整
```

### 第三步：设计算法

```
算法步骤：
1. 排序 g 和 s
2. 双指针：i 指向孩子，j 指向饼干
3. 遍历：
   - 如果 s[j] >= g[i]：满足，i++, j++, count++
   - 否则：饼干太小，换更大的 j++
4. 返回 count
```

---

## 解法：排序 + 双指针贪心

```java
public int findContentChildren(int[] g, int[] s) {
    // 排序
    Arrays.sort(g);
    Arrays.sort(s);
    
    int i = 0, j = 0;  // i:孩子指针, j:饼干指针
    int count = 0;
    
    while (i < g.length && j < s.length) {
        if (s[j] >= g[i]) {
            // 当前饼干能满足当前孩子
            count++;
            i++;  // 下一个孩子
            j++;  // 下一个饼干
        } else {
            // 饼干太小，换更大的
            j++;
        }
    }
    
    return count;
}
```

### 完整执行图解

```
示例1：g = [1,2,3], s = [1,1]

排序后：g = [1,2,3], s = [1,1]

========== 执行过程 ==========

初始：i=0(孩子1), j=0(饼干1), count=0

第1轮：s[0]=1 >= g[0]=1 ?
┌─────────────────────────────────────────┐
│  1 >= 1，满足！                          │
│  count = 1                               │
│  i = 1 (下一个孩子2)                      │
│  j = 1 (下一个饼干1)                      │
└─────────────────────────────────────────┘

第2轮：s[1]=1 >= g[1]=2 ?
┌─────────────────────────────────────────┐
│  1 >= 2，不满足                          │
│  饼干太小，换更大的                       │
│  j = 2 (没有更多饼干了)                   │
└─────────────────────────────────────────┘

j >= s.length，结束
返回 count = 1 ✓

===============================

示例2：g = [10,9,8,7], s = [5,6,7,8]

排序后：g = [7,8,9,10], s = [5,6,7,8]

初始：i=0, j=0, count=0

第1轮：s[0]=5 >= g[0]=7? 否，j=1
第2轮：s[1]=6 >= g[0]=7? 否，j=2
第3轮：s[2]=7 >= g[0]=7? 是，count=1, i=1, j=3
第4轮：s[3]=8 >= g[1]=8? 是，count=2, i=2, j=4

j >= s.length，结束
返回 count = 2 ✓

满足：胃口7的孩子得到饼干7，胃口8的孩子得到饼干8
```

## 贪心正确性证明

```
证明：为什么小饼干给小胃口是最优的？

反证法：
假设最优解中，存在一个小饼干给了大胃口，一个大饼干给了小胃口。
即：s_small >= g_big, s_big >= g_small

交换：
- 小饼干给小胃口：s_small >= g_small？
  因为 s_small >= g_big > g_small，所以满足
- 大饼干给大胃口：s_big >= g_big？
  因为 s_big >= s_small >= g_big，所以满足

交换后仍然满足，且满足人数不变。

结论：总可以通过交换，使得小饼干给小胃口，不影响最优性。
```

## 边界 Case

**Case 1: 没有饼干**

```
g = [1,2], s = []
返回 0 ✓
```

**Case 2: 没有孩子**

```
g = [], s = [1,2]
返回 0 ✓
```

**Case 3: 所有饼干都太小**

```
g = [5,6], s = [1,2,3]
所有饼干都小于5，无法满足任何孩子
返回 0 ✓
```

**Case 4: 所有饼干都足够大**

```
g = [1,2,3], s = [10,10,10]
能满足所有3个孩子
返回 3 ✓
```

**Case 5: 饼干比孩子多**

```
g = [1,2], s = [1,2,3,4]
能满足2个孩子
返回 2 ✓
```

## 复杂度分析

```
时间复杂度：O(n log n + m log m)
- 排序孩子：O(n log n)
- 排序饼干：O(m log m)
- 双指针遍历：O(n + m)

空间复杂度：O(1)
- 只使用了几个指针
```
