# TX 目录 - 贪心算法专题 (Greedy Algorithm)

这个目录帮你彻底理解贪心算法，从「什么是贪心」到「贪心的证明」。

## 为什么学贪心？

贪心算法是解决「最优选择」问题的高效方法：
- 区间调度问题（会议安排）
- 跳跃游戏
- 分发糖果
- 股票买卖
- 哈夫曼编码

贪心比动态规划更快（通常是 O(n) 或 O(n log n)），但难点在于**证明正确性**。

## 贪心算法的核心概念

### 什么是贪心？

```
贪心算法（Greedy Algorithm）：
- 每一步都做出当前看起来最优的选择
- 不回溯，不反悔
- 希望局部最优能导致全局最优

形象比喻：
- 爬山：每一步都往高处走，希望到达山顶
- 买东西：每次选性价比最高的
- 约会：每次都选眼前最好看的（误）

关键问题：
贪心一定能得到最优解吗？
答案：不一定！需要证明。
```

### 贪心 vs 动态规划

```
贪心：
- 每一步做最优选择，不回溯
- 时间复杂度低
- 需要证明正确性
- 适用：最优子结构 + 贪心选择性质

动态规划：
- 枚举所有可能，保存子问题解
- 时间复杂度高
- 一定能得到最优解
- 适用：最优子结构 + 重叠子问题

对比示例：
问题：有1元、3元、4元硬币，凑6元，最少几枚？

贪心：每次选最大面值
- 选4元，剩2元
- 选1元，剩1元
- 选1元，完成
- 结果：3枚（4+1+1）

实际最优：2枚（3+3）

结论：贪心失败！需要用动态规划。
```

### 贪心算法的适用条件

```
1. 最优子结构
   - 问题的最优解包含子问题的最优解

2. 贪心选择性质
   - 局部最优选择能导致全局最优

如何证明？
- 反证法：假设贪心不是最优，推出矛盾
- 归纳法：证明每一步贪心选择都是正确的
```

## 贪心算法的经典应用场景

### 场景1：区间调度问题

```
问题：给定多个区间，求最多能不重叠地选多少个

示例：[[1,3], [2,4], [3,5]]
- 选[1,3]和[3,5]，最多2个

贪心策略：
- 按结束时间排序
- 每次选结束最早的，且不与已选重叠的

为什么正确？
- 结束越早，留给后面的时间越多
- 证明：反证法，假设最优解第一个不是结束最早的，可以替换

过程：
1. 按结束时间排序：[1,3], [2,4], [3,5]
2. 选[1,3]，结束时间=3
3. [2,4]开始时间2<3，跳过
4. [3,5]开始时间3>=3，选
5. 结果：[1,3], [3,5]
```

### 场景2：跳跃游戏

```
问题：数组每个元素表示能跳的最远距离，能否到达最后一个？

示例：nums = [2,3,1,1,4]
- 从0能跳到1或2
- 从1能跳到2,3,4
- 能到达4

贪心策略：
- 维护当前能到达的最远位置 maxReach
- 遍历每个位置，更新 maxReach
- 如果 maxReach >= 最后位置，返回 true

过程：
i=0: nums[0]=2, maxReach=max(0, 0+2)=2
i=1: nums[1]=3, maxReach=max(2, 1+3)=4
     4 >= 4，返回 true ✓
```

### 场景3：分发糖果

```
问题：n个孩子站成一排，每个孩子有一个评分。分发糖果要求：
- 每个孩子至少1个
- 评分高的孩子比相邻的糖果多

示例：ratings = [1,0,2]
- 孩子0评分1，孩子1评分0，孩子2评分2
- 分配：[2,1,2]（满足条件）

贪心策略：
- 从左到右：如果右边评分高，糖果+1
- 从右到左：如果左边评分高，糖果取max(当前, 右边+1)

过程：
初始：[1,1,1]

从左到右：
- i=1: ratings[1]>ratings[0]? 0>1? 否
- i=2: ratings[2]>ratings[1]? 2>0? 是，candy[2]=candy[1]+1=2
结果：[1,1,2]

从右到左：
- i=1: ratings[1]>ratings[2]? 0>2? 否
- i=0: ratings[0]>ratings[1]? 1>0? 是，candy[0]=max(1, 1+1)=2
结果：[2,1,2]

总和：5 ✓
```

### 场景4：股票买卖 II

```
问题：可以买卖多次，求最大利润

示例：prices = [7,1,5,3,6,4]

贪心策略：
- 只要今天比昨天贵，就昨天买今天卖
- 把所有上涨的利润都赚到

过程：
7→1：跌，不买
1→5：涨，买在1卖在5，利润+4
5→3：跌，不买
3→6：涨，买在3卖在6，利润+3
6→4：跌，不买

总利润：4+3 = 7 ✓

为什么正确？
- 把连续上涨拆成每天买卖，利润相同
- [1,5,6]：5-1 + 6-5 = 6-1
```

### 场景5：根据身高重建队列

```
问题：每个人有身高h和前面人数k，重建队列

示例：[[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]

贪心策略：
- 按身高降序排列，身高相同按k升序
- 逐个插入，位置就是k

过程：
1. 排序：[[7,0],[7,1],[6,1],[5,0],[5,2],[4,4]]
2. 插入[7,0]：位置0 → [7,0]
3. 插入[7,1]：位置1 → [7,0],[7,1]
4. 插入[6,1]：位置1 → [7,0],[6,1],[7,1]
5. 插入[5,0]：位置0 → [5,0],[7,0],[6,1],[7,1]
6. 插入[5,2]：位置2 → [5,0],[7,0],[5,2],[6,1],[7,1]
7. 插入[4,4]：位置4 → [5,0],[7,0],[5,2],[6,1],[4,4],[7,1]

结果：[[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]] ✓

为什么正确？
- 先放高的，矮的插入不影响高的的k值
```

## 贪心算法的证明方法

```
1. 反证法
   - 假设贪心解不是最优解
   - 推出矛盾

2. 交换论证
   - 假设最优解和贪心解第一个不同位置
   - 交换后不会变差
   - 逐步把最优解变成贪心解

3. 归纳法
   - 证明第一步贪心选择正确
   - 证明每一步在第一步基础上继续贪心正确
```

## 贪心学习建议

1. **先判断能否贪心**：不是最优子结构的问题不能用
2. **多积累经典模型**：区间问题、跳跃游戏、分发糖果
3. **学会证明**：反证法和交换论证最常用
4. **对比动态规划**：贪心更快，但适用范围窄
