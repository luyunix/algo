# LB006 - 相交链表 题解

## 思路是怎么想出来的

### 第一步：明确问题

```
问题：找到两个链表的交点

示例：
listA: 4 → 1 → 8 → 4 → 5
              ↑
listB: 5 → 6 → 1

交点是值为8的节点（注意：交点是指针相同，不是值相同）
```

### 第二步：发现问题关键

```
关键洞察：
- 两个链表从交点开始，后面完全相同
- listA长度 = a + c（a是交点前，c是交点后）
- listB长度 = b + c（b是交点前，c是交点后）
- 如果能让两个指针走相同步数到达交点...

想法：
- 指针A走完listA，再从头走listB
- 指针B走完listB，再从头走listA
- 两个指针都走了 a + b + c 步，会在交点相遇
```

### 第三步：理解浪漫相遇法

```
比喻：
两个人从不同起点出发，速度相同
走到尽头后，从对方的起点重新开始
如果存在交点，他们一定会在交点相遇

数学证明：
- 指针A走的距离：a + c + b
- 指针B走的距离：b + c + a
- 两者相等，所以同时到达交点

如果不相交：
- 两个指针都走了 a + b 后，同时到达null
- 返回null
```

---

## 解法：双指针浪漫相遇法

```java
public ListNode getIntersectionNode(ListNode headA, ListNode headB) {
    // 如果任一链表为空，肯定不相交
    if (headA == null || headB == null) {
        return null;
    }
    
    // 初始化两个指针
    ListNode ptrA = headA;  // 指针A从链表A头开始
    ListNode ptrB = headB;  // 指针B从链表B头开始
    
    // 两个指针同时走，直到相遇
    while (ptrA != ptrB) {
        // 指针A走一步
        if (ptrA == null) {
            // A走到头了，从B的头重新开始
            ptrA = headB;
        } else {
            // 正常走一步
            ptrA = ptrA.next;
        }
        
        // 指针B走一步
        if (ptrB == null) {
            // B走到头了，从A的头重新开始
            ptrB = headA;
        } else {
            // 正常走一步
            ptrB = ptrB.next;
        }
    }
    
    // ptrA == ptrB时，就是交点（或都是null）
    return ptrA;
}
```

              ↑

初始：
ptrA在4（listA头）

## 不相交的情况

```
listA: 1 → 2 → 3
listB: 4 → 5

执行过程：
ptrA: 1 → 2 → 3 → null → 4 → 5 → null
ptrB: 4 → 5 → null → 1 → 2 → 3 → null

最后ptrA == ptrB == null，返回null ✓
```

## 关键点分析

### 为什么这样能相遇？

```
假设：
- listA交点前有a个节点
- listB交点前有b个节点
- 交点后有c个节点（包括交点）

指针A走的距离：a + c + b
指针B走的距离：b + c + a

两者相等！所以一定同时到达交点。
```

### 如果不用这个方法？

```
方法2：计算长度差
1. 先遍历两个链表，计算长度
2. 长的链表先走长度差步
3. 然后两个指针一起走，直到相遇

这个方法也可以，但需要两次遍历。
浪漫相遇法更优雅，只需一次遍历。
```

---

## 边界 Case

**Case 1: 一个链表为空**
```
headA = null
直接返回null ✓
```

**Case 2: 两个链表都不相交**
```
都走完，最后ptrA=ptrB=null
返回null ✓
```

**Case 3: 交点在链表头**
```
listA和listB从同一个节点开始
ptrA和ptrB一开始就相等
返回头节点 ✓
```

**Case 4: 交点在链表尾**
```
最后一个节点是交点
两个指针会在最后一个节点相遇 ✓
```

## 复杂度分析

```
时间复杂度：O(n + m)
- n是listA长度，m是listB长度
- 最坏情况下，每个指针走n+m步

空间复杂度：O(1)
- 只使用了两个指针
```
