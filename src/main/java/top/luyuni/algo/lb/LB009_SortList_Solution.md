# LB009 - 排序链表 题解

## 思路是怎么想出来的

### 第一步：明确问题

```
问题：对链表进行升序排序

示例：
输入：4 → 2 → 1 → 3
输出：1 → 2 → 3 → 4

要求：
- 时间复杂度 O(n log n)
- 空间复杂度尽可能低
```

### 第二步：选择排序算法

```
常见排序算法对比：

冒泡/选择/插入：O(n²)，太慢
快速排序：O(n log n)，但链表不方便随机访问
归并排序：O(n log n)，适合链表！
堆排序：O(n log n)，但需要随机访问

选择：归并排序
- 链表可以方便地找中点（快慢指针）
- 链表合并比数组简单（不需要额外空间）
```

### 第三步：归并排序思路

```
归并排序核心：分治法
1. 分：把链表分成两半
2. 治：递归排序左右两半
3. 合：合并两个有序链表

示例：4 → 2 → 1 → 3

分：
- 找中点：4 → 2 | 1 → 3
- 左半：4 → 2
- 右半：1 → 3

递归排序：
- 左半排序：2 → 4
- 右半排序：1 → 3

合并：
- 2 → 4 和 1 → 3 合并
- 结果：1 → 2 → 3 → 4
```

## 解法：归并排序

```java
public ListNode sortList(ListNode head) {
    // 递归终止条件：空链表或只有一个节点，已经有序
    if (head == null || head.next == null) {
        return head;
    }
    
    // 步骤1：找中点，把链表分成两半
    // 使用快慢指针，slow最后指向中点或左半部分末尾
    ListNode slow = head;           // 慢指针，每次走1步
    ListNode fast = head.next;      // 快指针，每次走2步（注意从next开始）
    
    // 快指针走到末尾时，慢指针在中间
    while (fast != null && fast.next != null) {
        slow = slow.next;           // 慢指针走1步
        fast = fast.next.next;      // 快指针走2步
    }
    
    // 现在slow指向左半部分末尾
    // mid是右半部分的头
    ListNode mid = slow.next;       // 右半部分的头
    slow.next = null;               // 断开左右两部分
    
    // 步骤2：递归排序左右两部分
    ListNode left = sortList(head);     // 排序左半部分
    ListNode right = sortList(mid);     // 排序右半部分
    
    // 步骤3：合并两个有序链表
    return merge(left, right);
}

// 合并两个有序链表
private ListNode merge(ListNode l1, ListNode l2) {
    // 创建虚拟头节点，方便处理
    ListNode dummy = new ListNode(0);
    ListNode tail = dummy;          // tail指向合并后链表的尾
    
    // 当两个链表都不为空时，比较并连接较小的节点
    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) {     // l1的值较小
            tail.next = l1;         // 连接l1
            l1 = l1.next;           // l1后移
        } else {                    // l2的值较小
            tail.next = l2;         // 连接l2
            l2 = l2.next;           // l2后移
        }
        tail = tail.next;           // tail后移
    }
    
    // 连接剩余的节点（l1或l2可能还有剩余）
    if (l1 != null) {
        tail.next = l1;             // 连接l1剩余部分
    }
    if (l2 != null) {
        tail.next = l2;             // 连接l2剩余部分
    }
    
    // 返回合并后的链表头（跳过虚拟头）
    return dummy.next;
}
```

### 完整执行图解

```
示例：4 → 2 → 1 → 3

========== 第1层递归：sortList(4→2→1→3) ==========

找中点：
slow=4, fast=2（fast从next开始）

第1轮：
┌─────────────────────────────────────────┐
│  slow走1步：4 → 2                        │
│  fast走2步：2 → 1 → 3，fast=1            │
└─────────────────────────────────────────┘

第2轮：
┌─────────────────────────────────────────┐
│  fast.next=3不为null                     │
│  fast.next.next=null，停止               │
│  slow停在2                               │
└─────────────────────────────────────────┘

分割：
左半：4 → 2（head到slow）
右半：1 → 3（slow.next开始）
断开：slow.next = null

========== 第2层递归左：sortList(4→2) ==========

找中点：
slow=4, fast=2
fast不为null，fast.next=null，停止
slow停在4

分割：
左半：4
右半：2

递归排序：
sortList(4) → 返回4（只有一个节点）
sortList(2) → 返回2（只有一个节点）

合并 merge(4, 2)：
┌─────────────────────────────────────────┐
│  dummy → ?                               │
│  4 > 2，连接2                            │
│  dummy → 2                               │
│  2后面连接4                              │
│  dummy → 2 → 4                           │
└─────────────────────────────────────────┘

返回 2 → 4

========== 第2层递归右：sortList(1→3) ==========

同理：
分割：左半1，右半3
合并 merge(1, 3)：
1 < 3，连接1，再连接3
返回 1 → 3

========== 回到第1层：合并 merge(2→4, 1→3) ==========

初始：l1=2→4, l2=1→3, dummy, tail=dummy

第1轮：
┌─────────────────────────────────────────┐
│  l1.val=2, l2.val=1                      │
│  1 < 2，连接l2                           │
│  tail.next = 1                           │
│  l2后移到3                               │
│  tail后移到1                             │
│  结果：dummy → 1                          │
└─────────────────────────────────────────┘

第2轮：
┌─────────────────────────────────────────┐
│  l1.val=2, l2.val=3                      │
│  2 < 3，连接l1                           │
│  tail.next = 2                           │
│  l1后移到4                               │
│  tail后移到2                             │
│  结果：dummy → 1 → 2                      │
└─────────────────────────────────────────┘

第3轮：
┌─────────────────────────────────────────┐
│  l1.val=4, l2.val=3                      │
│  3 < 4，连接l2                           │
│  tail.next = 3                           │
│  l2后移到null                            │
│  tail后移到3                             │
│  结果：dummy → 1 → 2 → 3                  │
└─────────────────────────────────────────┘

l2为null，连接l1剩余部分：
tail.next = 4

最终结果：dummy → 1 → 2 → 3 → 4

返回 1 → 2 → 3 → 4 ✓
```

## 关键点分析

### 为什么 fast 从 head.next 开始？

```
如果从 head 开始：
- 链表：1 → 2 → 3 → 4
- slow最后停在3，分割成 1→2 和 3→4
- 这是正确的

但链表：1 → 2 → 3
- slow最后停在2，分割成 1 和 2→3
- 这也是正确的

如果从 head.next 开始：
- 链表：1 → 2 → 3 → 4
- slow停在2，分割成 1→2 和 3→4

两种方式都可以，只是分割点不同。
```

### 递归终止条件

```
if (head == null || head.next == null)

- head == null：空链表
- head.next == null：只有一个节点

这两种情况都已经有序，直接返回。
```

---

## 边界 Case

**Case 1: 空链表**
```
head = null
返回null ✓
```

**Case 2: 单个节点**
```
1
返回1 ✓
```

**Case 3: 两个节点**
```
2 → 1
分割：2 和 1
合并：1 → 2 ✓
```

**Case 4: 已经有序**
```
1 → 2 → 3 → 4
正常排序，返回原顺序 ✓
```

**Case 5: 逆序**
```
4 → 3 → 2 → 1
排序后：1 → 2 → 3 → 4 ✓
```

## 复杂度分析

```
时间复杂度：O(n log n)
- 找中点：O(n)
- 递归深度：O(log n)
- 每层合并：O(n)
- 总：O(n log n)

空间复杂度：O(log n)
- 递归栈空间
- 不是O(1)，因为递归

进阶：自底向上归并排序可以达到O(1)空间
```
