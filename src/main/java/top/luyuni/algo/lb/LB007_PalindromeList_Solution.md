# LB007 - 回文链表 题解

## 思路是怎么想出来的

### 第一步：明确问题

```
问题：判断链表是否是回文

回文：正着读和反着读相同

示例：
1 → 2 → 2 → 1  是回文 ✓
1 → 2 → 3 → 2 → 1  是回文 ✓
1 → 2  不是回文 ✗
```

### 第二步：发现问题难点

```
难点：
- 链表不能倒着遍历（单向链表）
- 不能用额外空间存储所有值（要求O(1)空间）

想法：
- 找到链表的中点
- 反转后半部分
- 比较前半部分和反转后的后半部分
- 最后恢复链表（可选）
```

### 第三步：分步解决

```
步骤1：找中点（快慢指针）
- 慢指针每次走1步
- 快指针每次走2步
- 快指针到末尾时，慢指针在中点

步骤2：反转后半部分
- 从中点开始，反转链表

步骤3：比较
- 前半部分和反转后的后半部分逐个比较

步骤4：恢复（可选）
- 再次反转后半部分，恢复原链表
```

## 解法：找中点 + 反转 + 比较

```java
public boolean isPalindrome(ListNode head) {
    // 空链表或只有一个节点，是回文
    if (head == null || head.next == null) {
        return true;
    }
    
    // 步骤1：找中点（快慢指针）
    ListNode slow = head;  // 慢指针，每次走1步
    ListNode fast = head;  // 快指针，每次走2步
    
    // 快指针走到末尾时，慢指针在中点
    while (fast.next != null && fast.next.next != null) {
        slow = slow.next;           // 慢指针走1步
        fast = fast.next.next;      // 快指针走2步
    }
    
    // 现在slow指向中点（奇数长度）或前半部分末尾（偶数长度）
    // 步骤2：反转后半部分
    ListNode secondHalf = reverse(slow.next);
    
    // 步骤3：比较前半部分和后半部分
    ListNode firstHalf = head;      // 前半部分从头开始
    ListNode p2 = secondHalf;       // 后半部分从反转后的头开始
    
    boolean result = true;          // 假设是回文
    
    while (p2 != null) {            // 后半部分可能更短，以它为准
        if (firstHalf.val != p2.val) {  // 值不相等
            result = false;             // 不是回文
            break;                      // 跳出循环
        }
        firstHalf = firstHalf.next;     // 前半部分后移
        p2 = p2.next;                   // 后半部分后移
    }
    
    // 步骤4：恢复链表（可选，但推荐）
    slow.next = reverse(secondHalf);
    
    return result;
}

// 反转链表（标准写法）
private ListNode reverse(ListNode head) {
    ListNode prev = null;       // 前一个节点，初始为null
    ListNode curr = head;       // 当前节点，从head开始
    
    while (curr != null) {
        ListNode nextTemp = curr.next;  // 保存下一个节点
        curr.next = prev;               // 当前节点指向prev（反转）
        prev = curr;                    // prev后移
        curr = nextTemp;                // curr后移
    }
    
    return prev;                // prev是新头节点
}
```

### 完整执行图解

```
示例：1 → 2 → 2 → 1

========== 步骤1：找中点 ==========

初始：
slow在1，fast在1

第1轮：
┌─────────────────────────────────────────┐
│  slow走1步：1 → 2                       │
│  fast走2步：1 → 2 → 2                   │
└─────────────────────────────────────────┘

第2轮：
┌─────────────────────────────────────────┐
│  fast.next.next = null，停止            │
│  slow停在第2个节点（值为2）              │
└─────────────────────────────────────────┘

中点找到：slow指向第2个2

========== 步骤2：反转后半部分 ==========

后半部分：2 → 1（slow.next开始）

反转过程：
初始：prev=null, curr=2

第1轮：
┌─────────────────────────────────────────┐
│  nextTemp = 1（保存下一个）              │
│  2.next = null（指向prev）               │
│  prev = 2                                │
│  curr = 1                                │
└─────────────────────────────────────────┘
链表：2 → null

第2轮：
┌─────────────────────────────────────────┐
│  nextTemp = null                         │
│  1.next = 2（指向prev）                  │
│  prev = 1                                │
│  curr = null                             │
└─────────────────────────────────────────┘
链表：1 → 2 → null

反转完成：1 → 2

========== 步骤3：比较 ==========

前半部分：1 → 2
后半部分（反转后）：1 → 2

第1轮：
┌─────────────────────────────────────────┐
│  firstHalf.val = 1                       │
│  p2.val = 1                              │
│  1 == 1 ✓                                │
│  firstHalf后移，p2后移                    │
└─────────────────────────────────────────┘

第2轮：
┌─────────────────────────────────────────┐
│  firstHalf.val = 2                       │
│  p2.val = 2                              │
│  2 == 2 ✓                                │
│  firstHalf后移，p2后移                    │
│  p2 = null，结束                         │
└─────────────────────────────────────────┘

所有值都相等，是回文！返回true ✓
```

### 奇数长度示例

```
示例：1 → 2 → 3 → 2 → 1

找中点：
slow停在3（中间节点）

后半部分：2 → 1
反转后：1 → 2

比较：
前半部分：1 → 2（到slow为止，不包含3）
后半部分：1 → 2

相等，是回文 ✓

注意：中间的3不用比较
```

## 关键点分析

### 快慢指针找中点的细节

```
while (fast.next != null && fast.next.next != null)

偶数长度 1→2→2→1：
- 初始：slow=1, fast=1
- 第1轮：slow=2, fast=2
- fast.next=2, fast.next.next=null，停止
- slow在第2个2（前半部分末尾）

奇数长度 1→2→3→2→1：
- 初始：slow=1, fast=1
- 第1轮：slow=2, fast=3
- 第2轮：slow=3, fast=1（fast.next.next=null）
- slow在3（中间节点）

结论：slow总是指向中点或前半部分末尾
```

### 为什么要恢复链表？

```
实际面试中：
- 最好恢复链表，体现代码完整性
- 如果不恢复，可能会影响后续操作

恢复方法：
- 再次反转后半部分
- slow.next = reverse(secondHalf)
```

## 边界 Case

**Case 1: 空链表**
```
head = null
返回true ✓
```

**Case 2: 单个节点**
```
1
返回true ✓
```

**Case 3: 两个节点相同**
```
1 → 1
是回文 ✓
```

**Case 4: 两个节点不同**
```
1 → 2
不是回文 ✗
```

## 复杂度分析

```
时间复杂度：O(n)
- 找中点：O(n)
- 反转：O(n)
- 比较：O(n)
- 恢复：O(n)

空间复杂度：O(1)
- 只使用了几个指针
```
