# DL 目录 - 队列专题 (Queue)

这个目录帮你彻底理解队列，从「先进先出」到「队列的算法魔法」。

## 为什么学队列？

队列是解决「公平调度」和「广度优先搜索」的神器：
- 任务调度（CPU调度、线程池）
- BFS广度优先搜索（图的层序遍历）
- 缓存实现（LRU）
- 滑动窗口问题
- 生产者-消费者模型

## 队列的核心概念

### 什么是队列？

```
队列（Queue）是一种先进先出（FIFO）的数据结构：

想象排队买票：
- 新来的排到队尾
- 买完票的从队首离开
- 先来的先走，公平！

核心操作：
- offer/enqueue：入队，O(1)
- poll/dequeue：出队，O(1)
- peek：看队首，O(1)
- isEmpty：判空，O(1)
```

### 队列的直观理解

```
初始：空队列 []

offer(1): [1]
offer(2): [1, 2]  ← 2在队尾
offer(3): [1, 2, 3]

poll(): 返回1，队列变成 [2, 3]
poll(): 返回2，队列变成 [3]

peek(): 返回3，队列不变 [3]
```

### 队列 vs 栈

```
栈（Stack）：后进先出 LIFO
- 就像一叠盘子，最后放的最先拿
- 适合：递归、括号匹配、表达式求值

队列（Queue）：先进先出 FIFO
- 就像排队买票，先来的先走
- 适合：BFS、任务调度、滑动窗口

对比：
操作        栈          队列
插入        push        offer
删除        pop         poll
查看顶部    peek        peek
特点        后进先出    先进先出
```

## Java中的队列实现

```java
// 方式1：LinkedList（推荐）
Queue<Integer> queue = new LinkedList<>();

// 方式2：ArrayDeque（推荐，效率更高）
Queue<Integer> queue = new ArrayDeque<>();

// 常用操作
queue.offer(1);      // 入队（推荐，失败返回false）
queue.add(2);        // 入队（失败抛异常）
queue.poll();        // 出队（空返回null）
queue.remove();      // 出队（空抛异常）
queue.peek();        // 看队首（空返回null）
queue.element();     // 看队首（空抛异常）
queue.isEmpty();     // 判空
queue.size();        // 队列大小
```

## 双端队列 Deque

```
Deque（Double Ended Queue）：双端队列
- 可以在队首和队尾都进行插入和删除
- 既可以当栈用，也可以当队列用

Java实现：
Deque<Integer> deque = new ArrayDeque<>();

作为队列使用：
deque.offerLast(1);   // 队尾入队
deque.pollFirst();    // 队首出队

作为栈使用：
deque.offerLast(1);   // 入栈
deque.pollLast();     // 出栈
```

## 队列的经典应用场景

### 场景1：BFS广度优先搜索

```
问题：二叉树的层序遍历
示例：
    1
   / \
  2   3
 / \   \
4   5   6

输出：[[1], [2,3], [4,5,6]]

BFS思路：
1. 根节点入队
2. 每层节点数 = 当前队列大小
3. 依次出队，子节点入队
4. 重复直到队列为空

过程：
队列: [1]
第1层：出队1，子节点2,3入队，队列: [2, 3]
第2层：出队2,3，子节点4,5,6入队，队列: [4, 5, 6]
第3层：出队4,5,6，无子节点，队列: []
结束
```

### 场景2：滑动窗口最大值

```
问题：求数组每个滑动窗口的最大值
示例：nums = [1,3,-1,-3,5,3,6,7], k = 3
窗口：[1,3,-1] → [3,-1,-3] → [-1,-3,5] → ...
结果：[3, 3, 5, 5, 6, 7]

单调队列思路：
- 维护一个单调递减的双端队列
- 队列存的是索引，不是值
- 新元素从队尾入队，把前面比它小的都弹出
- 队首就是当前窗口最大值
- 队首过期（不在窗口内）就弹出

过程：
窗口 [1,3,-1]：
- 1入队，队列: [0(值1)]
- 3>1，弹出1，3入队，队列: [1(值3)]
- -1<3，-1入队，队列: [1(值3), 2(值-1)]
- 队首3是最大值 ✓

窗口 [3,-1,-3]：
- -3入队，队列: [1(3), 2(-1), 3(-3)]
- 队首3是最大值 ✓

窗口 [-1,-3,5]：
- 5>-3，弹出-3
- 5>-1，弹出-1
- 5>3，弹出3
- 5入队，队列: [4(5)]
- 队首5是最大值 ✓
```

### 场景3：生产者-消费者模型

```
问题：多线程环境下，生产者生产数据，消费者消费数据

队列作用：
- 作为缓冲区，解耦生产者和消费者
- 生产者满了就等待，消费者空了就等待
- 保证线程安全

Java实现：
BlockingQueue<Integer> queue = new LinkedBlockingQueue<>(100);

生产者线程：
while (true) {
    int data = produce();
    queue.put(data);  // 队列满时阻塞
}

消费者线程：
while (true) {
    int data = queue.take();  // 队列空时阻塞
    consume(data);
}
```

### 场景4：实现栈（用队列）

```
问题：只用队列实现栈

思路：
- 一个主队列，一个辅助队列
- push：直接入队
- pop：把前n-1个元素移到辅助队列，弹出最后一个

过程：
主队列: [1, 2, 3]，辅助队列: []

pop()：
- 1移到辅助，主: [2,3]，辅: [1]
- 2移到辅助，主: [3]，辅: [1,2]
- 弹出3，返回3
- 交换两个队列

主队列: [1, 2]，辅助队列: []
```
