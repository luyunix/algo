# DT004 - 零钱兑换 题解

## 状态定义是怎么想出来的

### 第一步：明确问题

```
问题：凑成金额amount，最少需要多少枚硬币
示例：coins = [1, 2, 5], amount = 11
答案：3枚（5+5+1）

暴力法：枚举所有可能的组合
- 每种硬币可以选0到amount/coin次
- 组合爆炸，需要DP
```

### 第二步：尝试直接定义

```
问题所求：最少硬币数
尝试：dp[i] = 凑成金额i所需的最少硬币数

验证是否可递推：
- 凑成11元，最后一枚硬币可能是1、2或5
- 如果是1：前面需要凑10元，用dp[10]枚
- 如果是2：前面需要凑9元，用dp[9]枚
- 如果是5：前面需要凑6元，用dp[6]枚
- 取最小值：dp[11] = min(dp[10], dp[9], dp[6]) + 1

可以递推！✓
```

### 第三步：确定状态转移

```
通用公式：
dp[i] = min(dp[i-coin] + 1) for all coin in coins

为什么加1？
- dp[i-coin]是凑前面金额的硬币数
- 加上当前这枚硬币，所以+1

为什么取min？
- 有多种硬币可选，选硬币数最少的方案
```

### 第四步：处理边界

```
dp[0] = 0（凑成0元需要0枚硬币）
dp[i]初始化为INF（表示暂时无法凑成）

如果dp[amount]还是INF，说明无法凑成，返回-1
```

## 核心模型：完全背包

每种硬币可以用**无限次**，这就是完全背包问题。

---

## 递推公式推导

**思考**：凑成金额 i，最后一枚硬币是什么？

```
假设 coins = [1, 2, 5]

凑成金额 11，最后一枚可能是：
- 1元：然后需要凑 10，用了 dp[10] 枚，总共 dp[10]+1
- 2元：然后需要凑 9，用了 dp[9] 枚，总共 dp[9]+1  
- 5元：然后需要凑 6，用了 dp[6] 枚，总共 dp[6]+1

取最小值：dp[11] = min(dp[10], dp[9], dp[6]) + 1
```

通用公式：

```
dp[i] = min(dp[i-coin] + 1) for all coin in coins
```

## 代码实现

```java
/**
 * 计算凑成指定金额所需的最少硬币数
 * @param coins 可用的硬币面额数组
 * @param amount 目标金额
 * @return 最少硬币数，如果无法凑成则返回-1
 */
public int coinChange(int[] coins, int amount) {
    // dp[i] = 凑成金额i所需的最少硬币数
    int[] dp = new int[amount + 1];

    // 初始化为无穷大（amount+1即可，因为最多用amount枚1元硬币）
    // 不能用Integer.MAX_VALUE，因为后面要+1，会溢出
    Arrays.fill(dp, amount + 1);

    // 凑成0元需要0枚硬币（关键初始条件）
    dp[0] = 0;

    // 遍历每个金额：从1元到amount元
    for (int i = 1; i <= amount; i++) {
        // 枚举每种硬币作为"最后一枚"加入的硬币
        for (int coin : coins) {
            // 只有当当前金额大于等于硬币面额时，才能使用这枚硬币
            if (i >= coin) {
                // 状态转移：
                // 1. dp[i] 保持原值（不用当前硬币）
                // 2. dp[i - coin] + 1（用当前硬币，前面金额的最优解+1）
                dp[i] = Math.min(dp[i], dp[i - coin] + 1);
            }
        }
    }

    // 如果dp[amount]还是初始值，说明没有任何组合能凑成该金额
    return dp[amount] > amount ? -1 : dp[amount];
}
```

## 边界 Case

**Case 1: 无法凑成**

```
coins = [2], amount = 3

dp = [0, 4, 4, 4]  // INF=4

i=1: 1 < 2，跳过，dp[1] = 4
i=2: dp[2] = min(4, dp[0]+1) = 1
i=3: 3 >= 2 ✓
     dp[3] = min(4, dp[1]+1) = min(4, 4+1) = 4

返回 dp[3] = 4 > 3，返回 -1 ✓
```

**Case 2: amount = 0**

```
直接返回 0
```

**Case 3: 只有1种硬币**

```
coins = [1], amount = 5

dp[1]=1, dp[2]=2, dp[3]=3, dp[4]=4, dp[5]=5
返回 5（用5枚1元）✓
```

## DP 四步法

| 步骤      | 内容         | 本题                                   |
|---------|------------|--------------------------------------|
| 1. 定义状态 | dp[i] 表示什么 | dp[i] = 凑成金额 i 所需的最少硬币数              |
| 2. 状态转移 | dp[i] 怎么算  | dp[i] = min(dp[i-coin] + 1) 对所有 coin |
| 3. 初始条件 | dp[0], 其他值 | dp[0]=0, 其他=INF                      |
| 4. 遍历顺序 | 从哪到哪       | 从 1 到 amount                         |

## 易错点详解

### 1. 为什么要初始化INF？

```java
// 错误做法
int[] dp = new int[amount + 1];  // 默认全是0

// 这样dp[i]永远是0，因为min(0, dp[i-coin]+1) = 0
```

**正确做法**：

```java
Arrays.fill(dp, amount +1);  // INF = amount + 1
```

### 2. 如何判断无解？

```java
return dp[amount]>amount ?-1:dp[amount];
```

如果 `dp[amount]` 还是初始值 `amount+1`，说明没有任何组合能凑成。

### 3. 为什么要判断 i >= coin？

防止数组越界！如果当前金额小于硬币面额，不能用这枚硬币。
