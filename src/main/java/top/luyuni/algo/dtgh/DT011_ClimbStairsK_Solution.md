# DT011 - 爬楼梯变种：每次最多K级 题解

## 状态定义是怎么想出来的

### 第一步：明确问题

```
问题：到第N级有多少种方法，每次可以迈1到K级
示例：N=4, K=3
答案：7种方法

暴力法：枚举所有可能的步数组合
- 组合爆炸，需要DP
```

### 第二步：类比经典爬楼梯

```
经典爬楼梯（K=2）：
- dp[i] = 到第i阶的方法数
- dp[i] = dp[i-1] + dp[i-2]

K级扩展：
- 最后一步可以迈1、2、...、K级
- 所以 dp[i] = dp[i-1] + dp[i-2] + ... + dp[i-K]
```

### 第三步：验证状态定义

```
定义：dp[i] = 到达第i级的方法数

验证递推：
到第i级，最后一步可能是：
- 从i-1迈1级：dp[i-1]种方法
- 从i-2迈2级：dp[i-2]种方法
- ...
- 从i-K迈K级：dp[i-K]种方法

所有情况相加：dp[i] = sum(dp[i-j]) for j in [1,K]

可以递推！✓
```

### 第四步：确定初始条件

```
dp[0] = 1（地面有一种方法：不动）
dp[i] = 0 for i < 0（越界，无方法）

答案：dp[N]
```

## 核心模型：爬楼梯的通用版

爬楼梯（K=2）的扩展，每次可以迈1到K级。

---

## 核心概念：dp[i] 是什么？

**dp[i] = 到达第 i 级台阶的方法数**（不是走了多少步！）

这是最容易混淆的地方。dp 数组存的是"有多少种不同的走法"，不是"走了多少级"。

---

## 递推公式推导

**技巧：从最后一步倒推**

假设要到达第 n 级，最后一步可能是：
- 从第 n-1 级迈 1 级上来
- 从第 n-2 级迈 2 级上来
- ...
- 从第 n-k 级迈 k 级上来

**为什么用 dp[n-1] 而不是 dp[n-1] + 1？**

因为 dp[n-1] 已经表示"到达第 n-1 级的所有方法数"，每种方法再迈 1 级到第 n 级，就贡献了 dp[n-1] 种方法到第 n 级。

**不是** "dp[n-1] 种方法 + 1 级台阶"，而是 "dp[n-1] 种方法，每种都再走 1 级"。

所以：
```
dp[n] = dp[n-1] + dp[n-2] + ... + dp[n-k]
```

## 代码实现

### 基础DP

```java
public int climbStairsK(int N, int K) {
    int[] dp = new int[N + 1];
    dp[0] = 1;  // 地面有一种方法
    
    for (int i = 1; i <= N; i++) {
        // 累加前K个值
        for (int j = 1; j <= K && i - j >= 0; j++) {
            dp[i] += dp[i - j];
        }
    }
    
    return dp[N];
}
```

### 空间优化（滑动窗口）

```java
public int climbStairsKOptimized(int N, int K) {
    if (N == 0) return 1;
    
    // 只保留前K个值
    int[] dp = new int[K];
    dp[0] = 1;  // dp[0]
    
    for (int i = 1; i <= N; i++) {
        // 当前值 = 前K个值的和
        int curr = 0;
        for (int j = 0; j < K && i - 1 - j >= 0; j++) {
            curr += dp[(i - 1 - j) % K];
        }
        dp[i % K] = curr;
    }
    
    return dp[N % K];
}
```

### 步长集合版本

```java
public int climbStairsWithSet(int N, int[] steps) {
    int[] dp = new int[N + 1];
    dp[0] = 1;
    
    for (int i = 1; i <= N; i++) {
        for (int step : steps) {
            if (i >= step) {
                dp[i] += dp[i - step];
            }
        }
    }
    
    return dp[N];
}
```

## 一步步走例子（N = 4, K = 3）

**问题**：到第4级，每次可以迈1、2或3级，有多少种方法？

### 手动列举验证

到达第4级的所有方法：
1. 1+1+1+1（每次迈1级）
2. 1+1+2（先迈1级，再迈1级，再迈2级）
3. 1+2+1
4. 2+1+1
5. 2+2
6. 1+3
7. 3+1

**共7种**，和dp计算结果一致。

### DP 计算过程

```
初始化：
dp[0] = 1  （地面有一种方法：不动）

i=1: dp[1] = dp[0] = 1
     解释：只能从 0 迈 1 级，有 1 种方法
     方法：[1]

i=2: dp[2] = dp[1] + dp[0] = 1 + 1 = 2
     解释：
     - 从 1 迈 1 级：dp[1]=1 种方法
     - 从 0 迈 2 级：dp[0]=1 种方法
     总共 2 种
     方法：[1,1], [2]

i=3: dp[3] = dp[2] + dp[1] + dp[0] = 2 + 1 + 1 = 4
     解释：
     - 从 2 迈 1 级：dp[2]=2 种方法（[1,1], [2]）
     - 从 1 迈 2 级：dp[1]=1 种方法（[1]）
     - 从 0 迈 3 级：dp[0]=1 种方法（不动）
     总共 4 种
     方法：[1,1,1], [2,1], [1,2], [3]

i=4: dp[4] = dp[3] + dp[2] + dp[1] = 4 + 2 + 1 = 7
     解释：
     - 从 3 迈 1 级：dp[3]=4 种方法
     - 从 2 迈 2 级：dp[2]=2 种方法
     - 从 1 迈 3 级：dp[1]=1 种方法
     总共 7 种 ✓
     方法：[1,1,1,1], [2,1,1], [1,2,1], [1,1,2], [2,2], [3,1], [1,3]
```

### 完整 DP 表

**完整 DP 表：**

| i | 0 | 1 | 2 | 3 | 4 |
|---|---|---|---|---|---|
| dp[i] | 1 | 1 | 2 | 4 | 7 |

计算过程：
i=0: dp[0] = 1
i=1: dp[1] = dp[0] = 1
i=2: dp[2] = dp[1] + dp[0] = 1 + 1 = 2
i=3: dp[3] = dp[2] + dp[1] + dp[0] = 2 + 1 + 1 = 4
i=4: dp[4] = dp[3] + dp[2] + dp[1] = 4 + 2 + 1 = 7 ✓

### 关键理解

**dp[2] = dp[1] + dp[0] = 2**
- dp[1]=1 表示到达第1级有1种方法（0→1）
- 这1种方法每种再迈1级到第2级，贡献1种
- dp[0]=1 表示到达第0级有1种方法（不动）
- 这1种方法迈2级到第2级，贡献1种
- **不是** "1种方法 + 1级 = 2"，而是 "1种 + 1种 = 2种"

## DP 四步法

| 步骤 | 内容 | 本题 |
|------|------|------|
| 1. 定义状态 | dp[i] 表示什么 | dp[i] = 到达第 i 级的方法数 |
| 2. 状态转移 | dp[i] 怎么算 | dp[i] = dp[i-1] + dp[i-2] + ... + dp[i-K] |
| 3. 初始条件 | dp[0] | dp[0] = 1 (地面有一种方法：不动) |
| 4. 遍历顺序 | 从哪到哪 | i: 1→N |

---

## 和爬楼梯的对比

| K值 | 递推公式 | 名称 |
|-----|----------|------|
| 2 | dp[i] = dp[i-1] + dp[i-2] | 斐波那契 |
| 3 | dp[i] = dp[i-1] + dp[i-2] + dp[i-3] | Tribonacci |
| K | dp[i] = dp[i-1] + ... + dp[i-K] | K-step |

## 边界 Case

**Case 1: N = 0**
```
已经在地面，算1种方法（不动）
返回 1 ✓
```

**Case 2: K = 1**
```
每次只能迈1级
只有一种方法：1+1+...+1
N级就有1种方法
返回 1 ✓
```

**Case 3: K >= N**
```
N=5, K=10
可以从地面直接迈到第5级
dp[5] = dp[4] + dp[3] + dp[2] + dp[1] + dp[0]
      = 8 + 4 + 2 + 1 + 1 = 16
实际上等于 2^(N-1) 当 K >= N 时
```

**Case 4: K = 2（普通爬楼梯）**
```
N=5, K=2
dp[0]=1, dp[1]=1, dp[2]=2, dp[3]=3, dp[4]=5, dp[5]=8
返回 8（斐波那契数列）✓
```
