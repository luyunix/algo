# DTGH 目录 - 动态规划专题 (Dynamic Programming - Too Hard)

这个目录帮你攻克动态规划，从"根本想不到递推公式"到"一眼看出状态转移"。

## 为什么动态规划难？

动态规划的难点不在于代码，而在于**思维转换**：
- 递归思维：从上往下想，f(n) 依赖 f(n-1)
- DP思维：从下往上想，先算 f(0), f(1)，再推 f(n)

**核心障碍**：
1. 不知道什么是「状态」
2. 想不出「状态转移方程」
3. 不确定「初始条件」

## 核心方法论

### 四步解题法

```
┌─────────────────────────────────────────┐
│  1. 定义状态：dp[i] 表示什么？            │
│  2. 找转移：dp[i] 如何从前面推导？        │
│  3. 初始化：dp[0], dp[1] 是什么？         │
│  4. 遍历顺序：从小到大还是从大到小？       │
└─────────────────────────────────────────┘
```

### 状态定义推导方法论

**为什么状态定义最难？**

因为状态定义不是「背出来的」，而是「推出来的」。以下是系统化的推导思路：

#### 第一步：明确问题所求

```
先不看DP，先想：如果不用DP，你会怎么暴力求解？

示例：爬楼梯
- 暴力法：枚举所有可能的爬法，计数
- 问题所求：到达第n阶的方法数
- 所以 dp[i] 应该存「方法数」
```

#### 第二步：尝试最直观的定义

```
根据问题直接定义：

爬楼梯：dp[i] = 到达第i阶的方法数  ✓ 这个可以
LIS：dp[i] = 前i个元素的最长递增子序列长度  ✗ 这个不行！

为什么LIS的不行？
- 前5个的最长长度是3，但不知道这个子序列结尾在哪
- 第6个元素能不能接上去？不知道！
- 信息不够，无法递推
```

#### 第三步：调整定义使其「可递推」

```
核心原则：状态定义必须包含「足够的信息」来推导下一个状态

LIS的修正：
- 原定义：dp[i] = 前i个的最长长度（信息不足）
- 新定义：dp[i] = 以第i个元素结尾的最长递增子序列长度

为什么新定义可以？
- 固定了结尾位置，就知道下一个元素能不能接
- nums[j] > nums[i] 就能接，dp[j] = dp[i] + 1
```

#### 第四步：验证「无后效性」

```
无后效性：未来的状态只依赖于当前状态，与如何到达当前状态无关

反例（有后效性）：
"dp[i] = 前i个元素选了哪些"
- 未来决策依赖于具体选了哪些，不只是数量
- 状态爆炸，无法DP

正例（无后效性）：
"dp[i] = 前i个元素的最大价值"
- 未来只需要知道这个最大价值，不需要知道具体选了什么
```

#### 状态定义自检清单

```
□ 这个定义能填表吗？（能从左到右依次计算）
□ 填完表能得出答案吗？（dp[n] 或 max(dp) 就是答案）
□ 状态转移自然吗？（推导过程符合直觉）
□ 满足无后效性吗？（未来只依赖当前状态值）
```

### 找递推公式的技巧

**技巧1：从最后一步倒推**

以爬楼梯为例：

```
问题：爬到第n阶有多少种方法？

从最后一步倒推：

第n阶 ← 第n-1阶（迈1步）
     ← 第n-2阶（迈2步）

所以：dp[n] = dp[n-1] + dp[n-2]

填表验证：
┌─────┬─────┬─────┬─────┬─────┐
│  n  │  0  │  1  │  2  │  3  │
├─────┼─────┼─────┼─────┼─────┤
│dp[n]│  1  │  1  │  2  │  3  │
└─────┴─────┴─────┴─────┴─────┘

计算过程：
dp[2] = dp[1] + dp[0] = 1 + 1 = 2
dp[3] = dp[2] + dp[1] = 2 + 1 = 3
```

**技巧2：画表格找规律**

以不同路径为例：

```
网格：3行3列，从左上到右下

dp[i][j] = 到达(i,j)的路径数

填表过程：
┌─────┬─────┬─────┐
│  1  │  1  │  1  │  ← 第一行只能从左来
├─────┼─────┼─────┤
│  1  │  2  │  3  │  ← (1,1)=1+1=2, (1,2)=2+1=3
├─────┼─────┼─────┤
│  1  │  3  │  6  │  ← (2,1)=1+2=3, (2,2)=3+3=6
└─────┴─────┴─────┘

规律：dp[i][j] = dp[i-1][j] + dp[i][j-1]
```

**技巧3：分类讨论**

以最大子数组和为例：

```
问题：以第i个元素结尾的最大子数组和

分类讨论：
┌─────────────────────────────────────┐
│ 情况1：前面连续                      │
│        dp[i] = dp[i-1] + nums[i]    │
├─────────────────────────────────────┤
│ 情况2：从i重新开始                   │
│        dp[i] = nums[i]              │
└─────────────────────────────────────┘

取最大值：
dp[i] = max(dp[i-1] + nums[i], nums[i])

填表（nums=[-2,1,-3,4,-1,2,1,-5,4]）：
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│  i  │  0  │  1  │  2  │  3  │  4  │  5  │  6  │  7  │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┤
│nums │ -2  │  1  │ -3  │  4  │ -1  │  2  │  1  │ -5  │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┤
│ dp  │ -2  │  1  │ -2  │  4  │  3  │  5  │  6  │  1  │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘

dp[3] = max(dp[2]+4, 4) = max(-2+4, 4) = 4  (重新开始)
dp[4] = max(dp[3]-1, -1) = max(4-1, -1) = 3 (连续)
```

### 空间优化技巧

**滚动数组**：如果 dp[i] 只依赖前几个状态，可以只用 O(1) 空间

```
以斐波那契为例：

原始：dp[i] = dp[i-1] + dp[i-2]
需要数组：dp[0...n]

优化：只保留前两个值
prev2, prev1, curr

过程：
初始: prev2=0, prev1=1
i=2: curr=prev2+prev1=1, prev2=1, prev1=1
i=3: curr=prev2+prev1=2, prev2=1, prev1=2
i=4: curr=prev2+prev1=3, prev2=2, prev1=3
...
```
