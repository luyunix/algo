## 01-DP基础专题

### 专题：适用问题

动态规划不是万能药，了解**什么时候用DP**比怎么用更重要。

**适用DP的问题特征：**

1. **最优子结构**：大问题的最优解包含小问题的最优解
2. **重叠子问题**：递归求解时会重复计算相同的子问题
3. **无后效性**：未来只依赖当前状态，与如何到达当前状态无关

**判断流程图：**

```
问题能否分解为子问题？
    │
    ├── 否 → 可能不是DP
    │
    └── 是 → 子问题是否重叠？
              │
              ├── 否 → 贪心或分治
              │
              └── 是 → 满足无后效性？
                        │
                        ├── 否 → 需要调整状态定义
                        └── 是 → 用DP！
```

**典型适用场景：**

| 场景 | 示例 |
|-----|------|
| 求最值 | 最大子数组和、最长递增子序列 |
| 求方案数 | 爬楼梯、路径计数 |
| 求可行性 | 能否凑成某金额、能否到达终点 |
| 求最小操作数 | 编辑距离、最小编辑代价 |

### 专题：解题步骤

**四步解题法：**

```
┌─────────────────────────────────────────────────────────┐
│  Step 1: 定义状态 - dp[i] 或 dp[i][j] 表示什么？         │
│  Step 2: 状态转移 - 如何从已知状态推导未知状态？          │
│  Step 3: 初始化 - 边界条件是什么？                       │
│  Step 4: 遍历顺序 - 从小到大还是从大到小？               │
└─────────────────────────────────────────────────────────┘
```

**Step 1: 状态定义**

状态定义是DP最难的部分。好的状态定义要满足：
- 能填表（能从前面的状态推导出来）
- 能得出答案（dp[n] 或 max(dp) 就是最终结果）
- 无后效性（未来只依赖当前状态值）

**Step 2: 状态转移方程**

找递推公式的技巧：
1. **从最后一步倒推**：思考到达当前状态的前一步可能是什么
2. **画表格找规律**：手动填几个小例子，观察规律
3. **分类讨论**：将所有可能情况列举出来，取最优

**Step 3: 初始化**

初始化的本质是定义边界条件：
- `dp[0]` 通常表示空集/初始状态
- 根据题意确定 `dp[1]`, `dp[2]` 等初始值

**Step 4: 遍历顺序**

- **正向遍历**：`dp[i]` 依赖 `dp[i-1]` 时用
- **反向遍历**：01背包问题中避免重复选取时用
- **二维遍历**：填表格时通常按行或按列遍历

### 专题：最值、可行、计数三种类型

**类型一：最值型DP**

问题特征：求最大/最小/最长等

```
状态转移：dp[i] = max/min(dp[i-1] + choice, choice)

典型题目：
- 最大子数组和
- 最长递增子序列 (LIS)
- 编辑距离
- 背包问题求最大价值
```

**类型二：可行型DP**

问题特征：判断是否可行（true/false）

```
状态转移：dp[i] = dp[i-1] && condition OR dp[i-2] && condition

典型题目：
- 跳跃游戏
- 单词拆分
- 正则表达式匹配
```

**类型三：计数型DP**

问题特征：求方案数、路径数

```
状态转移：dp[i] = dp[i-1] + dp[i-2] + ... (累加所有可能)

典型题目：
- 爬楼梯
- 不同路径
- 零钱兑换（求方案数）
```

### 专题：空间优化

**滚动数组优化**

当 `dp[i]` 只依赖前 k 个状态时，可以用 O(k) 空间替代 O(n)。

```java
// 原始版本：O(n) 空间
int[] dp = new int[n + 1];
dp[0] = 1; dp[1] = 1;
for (int i = 2; i <= n; i++) {
    dp[i] = dp[i-1] + dp[i-2];
}

// 优化版本：O(1) 空间
int prev2 = 1, prev1 = 1;
for (int i = 2; i <= n; i++) {
    int curr = prev1 + prev2;
    prev2 = prev1;
    prev1 = curr;
}
```

**二维转一维**

对于 `dp[i][j]`，如果只依赖上一行数据，可以用两行数组交替使用。

### 打家劫舍 & 股票买卖

这两类问题的共同点是：**状态与选择相关，当前选择影响后续选择**。

**打家劫舍问题：**

```

状态定义：dp[i] = 前i个房屋能偷的最大金额
状态转移：dp[i] = max(dp[i-1], dp[i-2] + nums[i])
```

```
核心约束：有冷冻期/交易次数限制
状态定义：

  - dp[i][1] = 第i天不持有股票的最大收益
状态转移：根据交易规则推导
```

### 爬楼梯问题

爬楼梯是最直观的入门模型，有多种变种：

**基础版本：**
- 每次可以爬1或2阶
- `dp[i] = dp[i-1] + dp[i-2]`

**扩展版本：**
- 每次可以爬1、2、...、k阶 → `dp[i] = dp[i-1] + dp[i-2] + ... + dp[i-k]`
- 带代价的爬楼梯 → 加上 `cost[i]`

### 匹配问题

字符串匹配类DP，通常涉及两个序列的比对。

**最长公共子序列（LCS）：**

```
状态定义：dp[i][j] = s1[0..i]和s2[0..j]的LCS长度
状态转移：
  - 若 s1[i] == s2[j]: dp[i][j] = dp[i-1][j-1] + 1
  - 否则: dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

**编辑距离：**

```
状态定义：dp[i][j] = 将s1[0..i]转换为s2[0..j]的最小操作数
状态转移：
  - 插入：dp[i][j] = dp[i][j-1] + 1
  - 删除：dp[i][j] = dp[i-1][j] + 1
  - 替换：dp[i][j] = dp[i-1][j-1] + (s1[i]==s2[j]?0:1)
```

| [`DT007_EditDistance.java`](DT007_EditDistance.java) | 编辑距离 | 双串DP |

## 03-特殊DP类别

### 树形DP

**适用场景**：问题定义在树上，父节点的状态依赖子节点的状态。

```
状态定义：dp[u] 表示以u为根的子树的状态
状态转移：dp[u] = f(dp[v1], dp[v2], ...) 其中v是u的子节点
遍历顺序：后序遍历（先处理子节点，再处理父节点）
```

**相关题目：**

| 文件 | 题目 | 说明 |
|-----|------|------|
| [`DT020_TreeDP.java`](DT020_TreeDP.java) | 打家劫舍 III | 二叉树上的最大独立集 |

**典型问题：**
- 树的最大独立集
- 树的直径
- 换根DP

### 区间DP

**适用场景**：问题涉及区间操作，大区间依赖小区间。

```
状态定义：dp[i][j] = 区间[i,j]的最优解
状态转移：枚举分割点k，dp[i][j] = max/min(dp[i][k] + dp[k+1][j] + cost)
遍历顺序：按区间长度从小到大枚举
```

**相关题目：**

| 文件 | 题目 | 说明 |
|-----|------|------|
| [`DT021_IntervalDP.java`](DT021_IntervalDP.java) | 最长回文子序列 | 区间DP经典题 |

**典型问题：**
- 石子合并
- 矩阵链乘法
- 回文子串
- 括号匹配

### 数位DP

**适用场景**：统计满足特定条件的数字个数（通常在某个范围内）。

```
状态定义：dp[pos][tight][leadingZero] = 从pos位开始填数的方案数
  - pos: 当前处理的数位位置
  - tight: 是否受限于上界（前几位是否贴着边界）
  - leadingZero: 是否有前导零
状态转移：枚举当前位填什么数字，递归处理下一位
```

| 文件 | 题目 | 说明 |
|-----|------|------|
| [`DT022_DigitDP.java`](DT022_DigitDP.java) | 数字 1 的个数 | 数位统计经典题 |

**典型问题：**
- 统计[l,r]中满足条件的数字个数
- 数字中包含特定模式

## 核心方法论

### 四步解题法

```
┌─────────────────────────────────────────┐
│  1. 定义状态：dp[i] 表示什么？            │
│  2. 找转移：dp[i] 如何从前面推导？        │
│  3. 初始化：dp[0], dp[1] 是什么？         │
│  4. 遍历顺序：从小到大还是从大到小？       │
└─────────────────────────────────────────┘
```

### 状态定义推导方法论

**为什么状态定义最难？**

因为状态定义不是「背出来的」，而是「推出来的」。以下是系统化的推导思路：

#### 第一步：明确问题所求

```
先不看DP，先想：如果不用DP，你会怎么暴力求解？

示例：爬楼梯
- 暴力法：枚举所有可能的爬法，计数
- 问题所求：到达第n阶的方法数
- 所以 dp[i] 应该存「方法数」
```

#### 第二步：尝试最直观的定义

```
根据问题直接定义：

爬楼梯：dp[i] = 到达第i阶的方法数  ✓ 这个可以
LIS：dp[i] = 前i个元素的最长递增子序列长度  ✗ 这个不行！

为什么LIS的不行？
- 前5个的最长长度是3，但不知道这个子序列结尾在哪
- 第6个元素能不能接上去？不知道！
- 信息不够，无法递推
```

#### 第三步：调整定义使其「可递推」

```
核心原则：状态定义必须包含「足够的信息」来推导下一个状态

LIS的修正：
- 原定义：dp[i] = 前i个的最长长度（信息不足）
- 新定义：dp[i] = 以第i个元素结尾的最长递增子序列长度

为什么新定义可以？
- 固定了结尾位置，就知道下一个元素能不能接
- nums[j] > nums[i] 就能接，dp[j] = dp[i] + 1
```

#### 第四步：验证「无后效性」

```
无后效性：未来的状态只依赖于当前状态，与如何到达当前状态无关

反例（有后效性）：
"dp[i] = 前i个元素选了哪些"
- 未来决策依赖于具体选了哪些，不只是数量
- 状态爆炸，无法DP

正例（无后效性）：
"dp[i] = 前i个元素的最大价值"
- 未来只需要知道这个最大价值，不需要知道具体选了什么
```

#### 状态定义自检清单

```
□ 这个定义能填表吗？（能从左到右依次计算）
□ 填完表能得出答案吗？（dp[n] 或 max(dp) 就是答案）
□ 状态转移自然吗？（推导过程符合直觉）
□ 满足无后效性吗？（未来只依赖当前状态值）
```

### 找递推公式的技巧

**技巧1：从最后一步倒推**

以爬楼梯为例：

```
问题：爬到第n阶有多少种方法？

从最后一步倒推：

第n阶 ← 第n-1阶（迈1步）
     ← 第n-2阶（迈2步）

所以：dp[n] = dp[n-1] + dp[n-2]

填表验证：
┌─────┬─────┬─────┬─────┬─────┐
│  n  │  0  │  1  │  2  │  3  │
├─────┼─────┼─────┼─────┼─────┤
│dp[n]│  1  │  1  │  2  │  3  │
└─────┴─────┴─────┴─────┴─────┘

计算过程：
dp[2] = dp[1] + dp[0] = 1 + 1 = 2
dp[3] = dp[2] + dp[1] = 2 + 1 = 3
```

**技巧2：画表格找规律**

以不同路径为例：

```
网格：3行3列，从左上到右下

dp[i][j] = 到达(i,j)的路径数

填表过程：
┌─────┬─────┬─────┐
│  1  │  1  │  1  │  ← 第一行只能从左来
├─────┼─────┼─────┤
│  1  │  2  │  3  │  ← (1,1)=1+1=2, (1,2)=2+1=3
├─────┼─────┼─────┤
│  1  │  3  │  6  │  ← (2,1)=1+2=3, (2,2)=3+3=6
└─────┴─────┴─────┘

规律：dp[i][j] = dp[i-1][j] + dp[i][j-1]
```

**技巧3：分类讨论**

以最大子数组和为例：

```
问题：以第i个元素结尾的最大子数组和

分类讨论：
┌─────────────────────────────────────┐
│ 情况1：前面连续                      │
│        dp[i] = dp[i-1] + nums[i]    │
├─────────────────────────────────────┤
│ 情况2：从i重新开始                   │
│        dp[i] = nums[i]              │
└─────────────────────────────────────┘

取最大值：
dp[i] = max(dp[i-1] + nums[i], nums[i])

填表（nums=[-2,1,-3,4,-1,2,1,-5,4]）：
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│  i  │  0  │  1  │  2  │  3  │  4  │  5  │  6  │  7  │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┤
│nums │ -2  │  1  │ -3  │  4  │ -1  │  2  │  1  │ -5  │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┤
│ dp  │ -2  │  1  │ -2  │  4  │  3  │  5  │  6  │  1  │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘

dp[3] = max(dp[2]+4, 4) = max(-2+4, 4) = 4  (重新开始)
dp[4] = max(dp[3]-1, -1) = max(4-1, -1) = 3 (连续)
```

### 空间优化技巧

**滚动数组**：如果 dp[i] 只依赖前几个状态，可以只用 O(1) 空间

```
以斐波那契为例：

原始：dp[i] = dp[i-1] + dp[i-2]
需要数组：dp[0...n]

优化：只保留前两个值
prev2, prev1, curr

过程：
初始: prev2=0, prev1=1
i=2: curr=prev2+prev1=1, prev2=1, prev1=1
i=3: curr=prev2+prev1=2, prev2=1, prev1=2
i=4: curr=prev2+prev1=3, prev2=2, prev1=3
...
```
