# DT001 - 斐波那契数列 题解

## 状态定义是怎么想出来的

### 第一步：明确问题

```
问题：求第 n 个斐波那契数
已知：F(0)=0, F(1)=1, F(n)=F(n-1)+F(n-2)

暴力法：直接递归
- fib(5) = fib(4) + fib(3)
- fib(4) = fib(3) + fib(2)
- ... 大量重复计算！
```

### 第二步：发现重叠子问题

```
画递归树：

        fib(5)
       /      \
   fib(4)    fib(3)
   /    \     /    \
fib(3) fib(2) fib(2) fib(1)
  |      |      |
fib(2) fib(1) fib(1)
  |
fib(1)

发现：fib(3) 算了2次，fib(2) 算了3次！
```

### 第三步：定义状态

```
思路：把计算过的结果存起来，避免重复计算

定义：dp[i] = 第 i 个斐波那契数

为什么这样定义？
- 问题问什么，dp就存什么（直接）
- 可以递推：dp[i] 只依赖 dp[i-1] 和 dp[i-2]
- 满足无后效性：算完 dp[i] 就不需要知道怎么算的
```

### 第四步：确定转移方程

```
根据定义：dp[i] = dp[i-1] + dp[i-2]

验证：
dp[2] = dp[1] + dp[0] = 1 + 0 = 1 ✓
dp[3] = dp[2] + dp[1] = 1 + 1 = 2 ✓
```

## 核心思想

DP 的本质：**用空间换时间，避免重复计算**

递归是从上往下算，DP是从下往上算。

## 代码实现

### 基础DP（数组版）

```java
/**
 * 计算第n个斐波那契数
 * @param n 斐波那契数的索引（从0开始）
 * @return 第n个斐波那契数
 */
public int fib(int n) {
    // ========== 步骤1：处理边界情况 ==========
    // 如果n是0或1，直接返回n本身
    // F(0) = 0, F(1) = 1
    if (n <= 1) return n;

    // ========== 步骤2：创建DP数组 ==========
    // dp数组的长度是n+1，因为我们要存dp[0]到dp[n]
    // dp[i]表示第i个斐波那契数
    int[] dp = new int[n + 1];

    // ========== 步骤3：设置初始条件 ==========
    // 根据定义，第0个斐波那契数是0
    dp[0] = 0;
    // 第1个斐波那契数是1
    dp[1] = 1;

    // ========== 步骤4：状态转移（递推计算） ==========
    // 从第2个开始，依次计算到第n个
    // i表示当前要计算的斐波那契数的索引
    for (int i = 2; i <= n; i++) {
        // 状态转移方程：F(i) = F(i-1) + F(i-2)
        // dp[i-1]是前一个斐波那契数
        // dp[i-2]是前两个斐波那契数
        dp[i] = dp[i - 1] + dp[i - 2];
    }

    // ========== 步骤5：返回结果 ==========
    // dp[n]就是我们要求的第n个斐波那契数
    return dp[n];
}
```

### 空间优化版

```java
/**
 * 空间优化版本：只用两个变量代替整个数组
 * 关键观察：计算dp[i]只需要dp[i-1]和dp[i-2]，不需要更早的值
 * 因此可以用滚动变量的方式，将空间复杂度从O(n)降到O(1)
 * @param n 斐波那契数的索引
 * @return 第n个斐波那契数
 */
public int fibOptimized(int n) {
    // ========== 步骤1：处理边界情况 ==========
    if (n <= 1) return n;

    // ========== 步骤2：初始化滚动变量 ==========
    // prev2代表dp[i-2]，初始值为dp[0] = 0
    int prev2 = 0;

    // prev1代表dp[i-1]，初始值为dp[1] = 1
    int prev1 = 1;

    // ========== 步骤3：迭代计算 ==========
    // 从第2个斐波那契数开始计算，直到第n个
    for (int i = 2; i <= n; i++) {
        // 计算当前的斐波那契数：curr = dp[i-1] + dp[i-2]
        int curr = prev1 + prev2;

        // 滚动更新：
        // prev2变成prev1（原来的dp[i-1]变成新的dp[i-2]）
        prev2 = prev1;

        // prev1变成curr（原来的dp[i]变成新的dp[i-1]）
        prev1 = curr;
    }

    // ========== 步骤4：返回结果 ==========
    // 循环结束后，prev1保存的就是dp[n]
    return prev1;
}
```

## DP 四步法回顾

| 步骤      | 内容           | 本题                        |
|---------|--------------|---------------------------|
| 1. 定义状态 | dp[i] 表示什么   | dp[i] = 第 i 项斐波那契数        |
| 2. 状态转移 | dp[i] 怎么算    | dp[i] = dp[i-1] + dp[i-2] |
| 3. 初始条件 | dp[0], dp[1] | dp[0]=0, dp[1]=1          |
| 4. 遍历顺序 | 从哪到哪         | 从 2 到 n                   |

## 关键洞察

**为什么DP比递归快？**

递归：

```
fib(5)
├── fib(4)
│   ├── fib(3)
│   │   ├── fib(2)
│   │   └── fib(1)
│   └── fib(2)
└── fib(3)  ← 重复计算！
    └── ...
```

DP：

```
dp[0] = 0
dp[1] = 1
dp[2] = dp[1] + dp[0] = 1  ← 每个只算一次
dp[3] = dp[2] + dp[1] = 2
dp[4] = dp[3] + dp[2] = 3
dp[5] = dp[4] + dp[3] = 5
```

**时间复杂度对比**：

- 递归：O(2^n)
- DP：O(n)
