# DT005 - 不同路径 题解

## 状态定义是怎么想出来的

### 第一步：明确问题

```
问题：从左上角到右下角有多少条不同路径
约束：每次只能向下或向右走

暴力法：枚举所有路径
- 需要走(m-1)次向下和(n-1)次向右
- 路径数 = C(m+n-2, m-1)，但用DP更通用
```

### 第二步：尝试定义状态

```
问题涉及二维网格，自然想到二维状态
尝试：dp[i][j] = 到达(i,j)的路径数

验证是否可递推：
- 到达(i,j)只能从上方来或从左方来
- 从上方来：有dp[i-1][j]条路径
- 从左方来：有dp[i][j-1]条路径
- 所以 dp[i][j] = dp[i-1][j] + dp[i][j-1]

可以递推！✓
```

### 第三步：确定初始条件

```
第一行：只能从左来，所以dp[0][j] = 1
第一列：只能从上下来，所以dp[i][0] = 1

dp[0][0] = 1（起点）
```

### 第四步：验证答案

```
最终答案就是dp[m-1][n-1]
因为状态定义直接就是「路径数」
```

## 核心模型：二维DP

从二维网格的左上角到右下角，每次只能向下或向右。

---

## 递推公式推导

**思考**：到达 (i, j) 的路径从哪来？

```
    (i-1,j)  ← 从上面来
        ↓
(i,j-1) → (i,j)
  ←从左面来
```

只有两种来源：

- 从上方 (i-1, j) 向下走
- 从左方 (i, j-1) 向右走

所以：

```
dp[i][j] = dp[i-1][j] + dp[i][j-1]
```

## 代码实现

### 二维DP

```java
public int uniquePaths(int m, int n) {
    int[][] dp = new int[m][n];
    
    // 第一行和第一列初始化为1
    for (int i = 0; i < m; i++) dp[i][0] = 1;
    for (int j = 0; j < n; j++) dp[0][j] = 1;
    
    // 填充其余格子
    for (int i = 1; i < m; i++) {
        for (int j = 1; j < n; j++) {
            dp[i][j] = dp[i-1][j] + dp[i][j-1];
        }
    }
    
    return dp[m-1][n-1];
}
```

### 空间优化（一维DP）

```java
public int uniquePathsOptimized(int m, int n) {
    int[] dp = new int[n];
    
    // 初始化为1（相当于第一行）
    Arrays.fill(dp, 1);
    
    // 从第二行开始
    for (int i = 1; i < m; i++) {
        for (int j = 1; j < n; j++) {
            // dp[j] 原来是上方，dp[j-1] 是左方（已更新）
            dp[j] = dp[j] + dp[j-1];
        }
    }
    
    return dp[n-1];
}
```

### 数学解法（组合数）

```java
public int uniquePathsMath(int m, int n) {
    // 总共需要走 (m-1) 次向下和 (n-1) 次向右
    // 从 (m+n-2) 步中选 (m-1) 步向下
    long result = 1;
    int k = Math.min(m - 1, n - 1);
    
    for (int i = 0; i < k; i++) {
        result = result * (m + n - 2 - i) / (i + 1);
    }
    
    return (int) result;
}
```

## 空间优化版执行图解

```
m=3, n=3

初始化：
dp = [1, 1, 1]  // 第一行

第1轮 i=1（计算第2行）：
j=1: dp[1] = dp[1] + dp[0] = 1 + 1 = 2
     dp = [1, 2, 1]
j=2: dp[2] = dp[2] + dp[1] = 1 + 2 = 3
     dp = [1, 2, 3]

第2轮 i=2（计算第3行）：
j=1: dp[1] = dp[1] + dp[0] = 2 + 1 = 3
     dp = [1, 3, 3]
j=2: dp[2] = dp[2] + dp[1] = 3 + 3 = 6
     dp = [1, 3, 6]

返回 dp[2] = 6 ✓
```

## 边界 Case

**Case 1: 1x1 网格**

```
m=1, n=1
直接返回 1（起点就是终点）✓
```

**Case 2: 1xN 网格**

```
m=1, n=5
只有一条路径：右右右右
返回 1 ✓
```

**Case 3: Mx1 网格**

```
m=5, n=1
只有一条路径：下下下下
返回 1 ✓
```

**Case 4: 2x2 网格**

```
m=2, n=2
dp:
1 1
1 2
返回 2 ✓
路径：右下、下右
```

## DP 四步法

| 步骤      | 内容            | 本题                                 |
|---------|---------------|------------------------------------|
| 1. 定义状态 | dp[i][j] 表示什么 | dp[i][j] = 到达 (i,j) 的路径数           |
| 2. 状态转移 | dp[i][j] 怎么算  | dp[i][j] = dp[i-1][j] + dp[i][j-1] |
| 3. 初始条件 | 第一行和第一列       | 都是 1                               |
| 4. 遍历顺序 | 从哪到哪          | i: 1→m-1, j: 1→n-1                 |

---

## 空间优化原理

```
计算第i行时，只需要：
- 第i-1行的值（上方）
- 第i行已经计算的值（左方）

所以用一个数组就够了：
dp[j] = dp[j]（原来的值，上方）+ dp[j-1]（新值，左方）
```
