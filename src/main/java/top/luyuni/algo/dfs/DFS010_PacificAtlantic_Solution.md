# DFS010 - 太平洋大西洋水流问题 题解

## 思路是怎么想出来的

### 第一步：明确问题

```
问题：给定m×n矩阵，表示每个单元格的高度
- 水可以从高流向低（或相等）
- 左边和上边是太平洋
- 右边和下边是大西洋
- 求能同时流向两个海洋的单元格

示例：
heights = [
  [1,2,2,3,5],
  [3,2,3,4,4],
  [2,4,5,3,1],
  [6,7,1,4,5],
  [5,1,1,2,4]
]

输出：[[0,4],[1,3],[1,4],[2,2],[3,0],[3,1],[4,0]]
```

### 第二步：逆向思维

```
正向思维：从每个单元格出发，看能否流到两个海洋
- 需要对每个单元格做DFS
- 时间复杂度高

逆向思维：从海洋出发，看能流到哪些单元格
- 从太平洋边界（左、上）开始，逆流而上
- 从大西洋边界（右、下）开始，逆流而上
- 两个海洋都能到达的单元格就是答案

为什么逆向可行？
- 水从A流到B（A>=B）
- 逆向：从B可以流到A（B<=A）
- 条件反过来即可
```

### 第三步：设计算法

```
算法步骤：
1. 从太平洋边界（第一行、第一列）开始DFS
   - 标记所有能到达的单元格为pacificReachable
2. 从大西洋边界（最后一行、最后一列）开始DFS
   - 标记所有能到达的单元格为atlanticReachable
3. 遍历所有单元格，找出两个都能到达的

DFS条件：
- 正向：当前高度 >= 邻居高度（水往低处流）
- 逆向：当前高度 <= 邻居高度（从低往高走）
```

## 解法：多源DFS（逆向思维）

```java
public List<List<Integer>> pacificAtlantic(int[][] heights) {
    List<List<Integer>> result = new ArrayList<>();
    if (heights == null || heights.length == 0) return result;
    
    int m = heights.length, n = heights[0].length;
    boolean[][] pacific = new boolean[m][n];   // 太平洋能到达
    boolean[][] atlantic = new boolean[m][n];  // 大西洋能到达
    
    // 从太平洋边界开始DFS
    for (int i = 0; i < m; i++) {
        dfs(heights, pacific, i, 0);      // 第一列
    }
    for (int j = 0; j < n; j++) {
        dfs(heights, pacific, 0, j);      // 第一行
    }
    
    // 从大西洋边界开始DFS
    for (int i = 0; i < m; i++) {
        dfs(heights, atlantic, i, n - 1); // 最后一列
    }
    for (int j = 0; j < n; j++) {
        dfs(heights, atlantic, m - 1, j); // 最后一行
    }
    
    // 找两个都能到达的
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (pacific[i][j] && atlantic[i][j]) {
                result.add(Arrays.asList(i, j));
            }
        }
    }
    
    return result;
}

private void dfs(int[][] heights, boolean[][] reachable, int i, int j) {
    int m = heights.length, n = heights[0].length;
    reachable[i][j] = true;
    
    // 四个方向
    int[][] dirs = {{-1,0}, {1,0}, {0,-1}, {0,1}};
    
    for (int[] dir : dirs) {
        int ni = i + dir[0], nj = j + dir[1];
        
        // 边界检查 && 未访问 && 能流过去（逆流：当前<=邻居）
        if (ni >= 0 && ni < m && nj >= 0 && nj < n 
            && !reachable[ni][nj] 
            && heights[ni][nj] >= heights[i][j]) {
            dfs(heights, reachable, ni, nj);
        }
    }
}
```

## 逆向思维的优势

```
正向：从每个单元格出发
- 时间：O(mn × mn) = O(m²n²)
- 每个单元格都要做一次DFS

逆向：从海洋边界出发
- 时间：O(mn)
- 只做两次DFS（太平洋、大西洋各一次）
- 每个单元格最多被访问两次

这就是多源BFS/DFS的思想！
```

## 边界 Case

**Case 1: 1×1矩阵**

```
[[1]]
同时接触太平洋和大西洋
返回[[0,0]] ✓
```

**Case 2: 1×n矩阵**

```
[[1,2,3]]
所有单元格都能流到两个海洋
返回[[0,0],[0,1],[0,2]] ✓
```

**Case 3: n×1矩阵**

```
[[1],[2],[3]]
类似1×n，所有都能流到两个海洋 ✓
```

**Case 4: 严格递减矩阵**

```
[[5,4,3],
 [4,3,2],
 [3,2,1]]

只有边界能流到海洋
内部单元格(1,1)=3，只能流向更低的地方
但边界已经是最低，流不出去
```
