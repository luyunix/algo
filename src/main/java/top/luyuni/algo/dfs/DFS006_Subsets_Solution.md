# DFS006 - 子集 题解

## 思路是怎么想出来的

### 第一步：明确问题

```
问题：给定数组[1,2,3]，求所有子集

子集：从数组中选取0个或多个元素（不重复选）

[1,2,3]的所有子集：
- 选0个：[]
- 选1个：[1], [2], [3]
- 选2个：[1,2], [1,3], [2,3]
- 选3个：[1,2,3]

共8个子集（2^3）
```

### 第二步：发现规律

```
关键洞察：每个元素有两种选择 - 选 或 不选

决策树：
        []
       / | \
     选1 选2 选3
     /     \
   [1]      []
   / \      / \
 选2 不选2 选2 不选2
 /     \    /     \
...    ...  ...    ...

这是一个典型的DFS+回溯问题！
```

### 第三步：设计算法

```
需要维护：
1. 当前路径（path）：已经选择的元素
2. 起始索引（start）：从哪个位置开始选（避免重复）

DFS步骤：
1. 把当前path加入结果（每个节点都是子集）
2. 从start开始遍历：
   a. 选择nums[i]，加入path
   b. DFS递归，start=i+1（从下一个开始，避免重复）
   c. 回溯：撤销选择

关键：每个节点都是有效子集，都要加入结果
```

---

## 解法：DFS + 回溯

```java
public List<List<Integer>> subsets(int[] nums) {
    // 结果列表，存储所有子集
    List<List<Integer>> result = new ArrayList<>();
    
    // 当前路径，存储当前正在构建的子集
    List<Integer> path = new ArrayList<>();
    
    // 从索引0开始DFS
    dfs(nums, 0, path, result);
    
    // 返回所有子集
    return result;
}

/**
 * DFS递归函数
 * @param nums 原数组
 * @param start 从哪个索引开始选择
 * @param path 当前路径（已选择的元素）
 * @param result 结果列表
 */
private void dfs(int[] nums, int start, 
                 List<Integer> path, List<List<Integer>> result) {
    // 关键：每个节点都是子集，加入结果
    // 新建ArrayList是因为path会被修改
    result.add(new ArrayList<>(path));
    
    // 从start开始，尝试每个元素
    for (int i = start; i < nums.length; i++) {
        // 做选择：把nums[i]加入当前路径
        path.add(nums[i]);
        
        // 递归，从i+1开始（避免重复选择）
        // 比如选了1之后，只能从2,3中选，不能再选1
        dfs(nums, i + 1, path, result);
        
        // 撤销选择（回溯）：移除最后一个元素
        // 这样path就恢复到选择nums[i]之前的状态
        path.remove(path.size() - 1);
    }
}
```

## 与全排列的区别

```
全排列：
- 每个元素只能用一次
- 路径长度固定（等于数组长度）
- 需要used数组标记
- 叶子节点才是结果

子集：
- 每个元素可选可不选
- 路径长度不固定
- 不需要used数组，用start索引控制
- 每个节点都是结果

关键区别：
- 全排列：dfs(nums, used, path, result)
- 子集：dfs(nums, start, path, result)
```

## 关键点分析

### 为什么从start开始而不是0？

```
如果从0开始：
- 选了1之后，还能再选1
- 会产生重复子集：[1,1], [1,2,1]...

从start开始：
- 选了1（索引0），下次从1（索引1）开始
- 只能从2,3中选，不能回头选1
- 避免重复
```

### 为什么每个节点都要加入结果？

```
子集的特点：
- [] 是子集（根节点）
- [1] 是子集（选了1之后）
- [1,2] 是子集（选了1,2之后）
- ...

每个DFS节点都对应一个子集
所以每个节点都要加入结果
```

## 边界 Case

**Case 1: 空数组**
```
nums = []
只有一个子集：[]
返回[[]] ✓
```

**Case 2: 只有一个元素**
```
nums = [1]
子集：[], [1]
返回[[], [1]] ✓
```

**Case 3: 两个元素**
```
nums = [1, 2]
子集：[], [1], [2], [1,2]
返回4个子集 ✓
```

**Case 4: 有重复元素（本题不考虑）**
```
nums = [1, 2, 2]
需要去重，本题假设无重复元素
```

## 复杂度分析

```
时间复杂度：O(n × 2^n)
- 共有2^n个子集
- 每个子集复制到结果需要O(n)

空间复杂度：O(n)
- 递归栈深度最多n
- path最多n个元素
```
