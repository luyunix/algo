# ZHAN002 - 最小栈 题解

## 思路是怎么想出来的

### 第一步：明确问题

```
问题：实现一个栈，支持O(1)时间获取最小值

普通栈的操作：
- push: O(1)
- pop: O(1)
- top: O(1)

新增要求：
- getMin: O(1)

难点：最小值会变化！
- push更小的数，最小值改变
- pop掉最小值，最小值也改变
```

### 第二步：如何跟踪最小值

```
方案1：用一个变量存最小值
- push时更新min
- 但pop时不知道之前的最小值是什么！
- 不可行

方案2：每个节点存当前最小值
- 每个元素入栈时，同时记录当前栈的最小值
- 这样pop时，栈顶就是之前的最小值
- 可行！

方案3：辅助栈（更优）
- 主栈正常存数据
- 辅助栈存最小值
- 辅助栈栈顶就是当前最小值
```

### 第三步：辅助栈的设计

```
辅助栈规则：
- push时：如果新元素 <= 辅助栈顶，也push到辅助栈
- pop时：如果弹出元素 == 辅助栈顶，辅助栈也pop
- getMin：返回辅助栈顶

为什么用 <= 而不是 < ？
- 处理重复最小值的情况
- 如 push(0), push(0), pop()
- 第一个0pop后，最小值还是0
```

---

## 解法：辅助栈

```java
class MinStack {
    private Deque<Integer> stack;      // 主栈
    private Deque<Integer> minStack;   // 辅助栈（存最小值）
    
    public MinStack() {
        stack = new ArrayDeque<>();
        minStack = new ArrayDeque<>();
    }
    
    public void push(int val) {
        stack.push(val);
        // 辅助栈为空 或 新元素 <= 当前最小值，入辅助栈
        if (minStack.isEmpty() || val <= minStack.peek()) {
            minStack.push(val);
        }
    }
    
    public void pop() {
        int val = stack.pop();
        // 如果弹出的是最小值，辅助栈也pop
        if (val == minStack.peek()) {
            minStack.pop();
        }
    }
    
    public int top() {
        return stack.peek();
    }
    
    public int getMin() {
        return minStack.peek();
    }
}
```

## 边界 Case

**Case 1: 重复最小值**

```
push(0), push(0), getMin(), pop(), getMin()

push(0): 主[0], 辅[0]
push(0): 0<=0，主[0,0], 辅[0,0]
getMin(): 0
pop(): 弹出0，辅也弹出，主[0], 辅[0]
getMin(): 0 ✓
```

**Case 2: 递减序列**

```
push(3), push(2), push(1)

主: [1,2,3], 辅: [1,2,3]
每次push都更小，都入辅助栈
```

**Case 3: 递增序列**

```
push(1), push(2), push(3)

主: [3,2,1], 辅: [1]
只有第一个入辅助栈
```

## 记忆口诀

```
最小栈用双栈存，主栈存数辅存小
push时看大小，小于等于辅也存
pop时看相等，相等辅栈也弹出
getMin看辅顶，O1时间取最小
```
