# DUI 目录 - 堆专题 (Heap)

这个目录帮你彻底理解堆，从「什么是堆」到「堆的实战应用」。

## 为什么学堆？

堆是解决「Top K问题」和「动态找最值」的神器：

- 求数组中第K大的元素
- 合并K个有序数组
- 任务调度（优先级队列）
- 图算法中的Dijkstra最短路径

## 堆的核心概念

### 什么是堆？

```
堆是一个完全二叉树，满足堆性质：
- 大顶堆：父节点 >= 子节点
- 小顶堆：父节点 <= 子节点

关键特性：
1. 完全二叉树：除了最后一层，其他层都是满的，最后一层从左到右填充
2. 堆顶元素一定是最大/最小值
3. 插入和删除都是 O(log n)
```

### 堆的存储方式

```
完全二叉树可以用数组存储，不需要指针！

对于索引 i 的节点：
- 父节点索引：(i-1) / 2
- 左子节点索引：2*i + 1
- 右子节点索引：2*i + 2

示例：大顶堆 [9, 8, 5, 6, 7, 2, 1]

        9          ← 索引0
       / \
      8   5        ← 索引1, 2
     / \  / \
    6  7 2   1     ← 索引3, 4, 5, 6

验证堆性质：
- 8的父节点：(1-1)/2 = 0，即9，9 >= 8 ✓
- 6的父节点：(3-1)/2 = 1，即8，8 >= 6 ✓
```

### 堆的核心操作

#### 1. 上浮（Sift Up / Swim）

```
场景：新元素插入到堆底，需要调整到正确位置

过程：
1. 新元素放到数组末尾
2. 与父节点比较
3. 如果不满足堆性质，交换
4. 重复直到满足堆性质或到达根

示例：小顶堆插入 3
原堆：[2, 5, 4, 9, 8]（对应树：2是根，5和4是子节点）

插入3：[2, 5, 4, 9, 8, 3]
         2
        / \
       5   4
      / \  /
     9  8 3

3的父节点是4，3 < 4，交换
[2, 5, 3, 9, 8, 4]
         2
        / \
       5   3
      / \  /
     9  8 4

3的父节点是2，3 > 2，停止 ✓
```

#### 2. 下沉（Sift Down / Sink）

```
场景：删除堆顶后，把堆底元素放到堆顶，需要调整到正确位置

过程：
1. 堆顶元素与堆底交换
2. 删除堆底（原堆顶）
3. 新堆顶与较大的子节点比较（大顶堆）
4. 如果不满足堆性质，交换
5. 重复直到满足堆性质或到达叶子

示例：大顶堆删除堆顶
原堆：[9, 8, 5, 6, 7]

Step 1：9和7交换
[7, 8, 5, 6, 9]，删除9 → [7, 8, 5, 6]

Step 2：7下沉
7的子节点是8和5，最大的是8
7 < 8，交换
[8, 7, 5, 6]

Step 3：继续下沉
7的子节点是6，7 > 6，停止 ✓

最终：[8, 7, 5, 6]
```

## Java中的堆实现

### PriorityQueue（优先队列）

```java
// 默认是小顶堆
PriorityQueue<Integer> minHeap = new PriorityQueue<>();

// 大顶堆
PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);

// 常用操作
minHeap.offer(5);      // 插入，O(log n)
minHeap.poll();        // 删除堆顶，O(log n)
minHeap.peek();        // 查看堆顶，O(1)
minHeap.size();        // 堆大小
```

### 手写堆（面试常考）

```java
class MinHeap {
    private int[] heap;
    private int size;
    
    public MinHeap(int capacity) {
        heap = new int[capacity];
        size = 0;
    }
    
    // 插入
    public void offer(int val) {
        heap[size] = val;
        siftUp(size);
        size++;
    }
    
    // 上浮
    private void siftUp(int i) {
        while (i > 0) {
            int parent = (i - 1) / 2;
            if (heap[parent] <= heap[i]) break;
            swap(parent, i);
            i = parent;
        }
    }
    
    // 删除堆顶
    public int poll() {
        int res = heap[0];
        heap[0] = heap[size - 1];
        size--;
        siftDown(0);
        return res;
    }
    
    // 下沉
    private void siftDown(int i) {
        while (2 * i + 1 < size) {
            int left = 2 * i + 1;
            int right = 2 * i + 2;
            int min = left;
            
            if (right < size && heap[right] < heap[left]) {
                min = right;
            }
            
            if (heap[i] <= heap[min]) break;
            swap(i, min);
            i = min;
        }
    }
}
```

## 堆的经典应用场景

### 场景1：Top K 问题

```
求数组中第K大的元素

思路：
1. 维护一个大小为K的小顶堆
2. 遍历数组，如果元素比堆顶大，替换堆顶
3. 最后堆顶就是第K大的元素

为什么是小顶堆？
- 堆顶是这K个中最小的
- 如果有更大的元素进来，替换堆顶
- 最后堆里是K个最大的，堆顶是第K大
```

### 场景2：合并K个有序数组

```
K个有序数组合并成一个有序数组

思路：
1. 把每个数组的第一个元素放入小顶堆
2. 每次取出堆顶（最小），放入结果
3. 从该元素所在的数组取下一个放入堆
4. 重复直到堆为空

时间复杂度：O(N log K)，N是总元素数
```

### 场景3：数据流中位数

```
数据流中不断有数字进来，实时求中位数

思路：
1. 维护两个堆：大顶堆（存较小的一半）+ 小顶堆（存较大的一半）
2. 保持两个堆的大小差不超过1
3. 中位数：
   - 如果大小相等，(大顶堆顶 + 小顶堆顶) / 2
   - 如果不等，元素多的那个堆顶

示例：
进来 1：小顶堆=[1]，大顶堆=[]，中位数=1
进来 2：小顶堆=[2]，大顶堆=[1]，中位数=(1+2)/2=1.5
进来 3：小顶堆=[2,3]，大顶堆=[1]，中位数=2
进来 4：小顶堆=[3,4]，大顶堆=[2,1]，中位数=(2+3)/2=2.5
```

## 堆学习建议

1. **先理解概念**：完全二叉树、堆性质、上浮下沉
2. **再掌握API**：PriorityQueue的使用
3. **最后手写堆**：面试常考，理解原理
4. **多做Top K题**：堆最常见的应用场景
