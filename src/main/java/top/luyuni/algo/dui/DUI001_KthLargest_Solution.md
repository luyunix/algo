# DUI001 - 数组中的第K个最大元素 题解

## 思路是怎么想出来的

### 第一步：明确问题

```
问题：找数组中第K大的元素
示例：nums = [3,2,1,5,6,4], k = 2
排序后：[6,5,4,3,2,1]，第2大的是5

暴力法：排序后取第K个
- 时间复杂度：O(n log n)
- 可以优化到O(n log k)
```

### 第二步：利用堆的性质

```
关键洞察：
- 小顶堆的堆顶是最小元素
- 如果维护一个大小为K的小顶堆
- 堆顶就是K个元素中最小的，也就是第K大的

为什么不是大顶堆？
- 大顶堆堆顶是最大元素
- 但我们不知道第K大是多少，无法直接找
- 小顶堆可以淘汰小的，保留大的
```

### 第三步：具体流程

```
维护大小为K的小顶堆：
1. 遍历数组，元素入堆
2. 如果堆大小>K，弹出堆顶（最小的）
3. 最后堆顶就是第K大的

示例：[3,2,1,5,6,4], k=2

遍历3：堆[3]
遍历2：堆[2,3]（2在堆顶）
遍历1：堆[1,3,2]，大小=3>2，弹出1，堆[2,3]
遍历5：堆[2,3,5]，大小=3>2，弹出2，堆[3,5]
遍历6：堆[3,5,6]，大小=3>2，弹出3，堆[5,6]
遍历4：堆[4,5,6]，大小=3>2，弹出4，堆[5,6]

堆顶是5，第2大的元素 ✓
```

---

## 解法：小顶堆

```java
public int findKthLargest(int[] nums, int k) {
    // 小顶堆
    PriorityQueue<Integer> minHeap = new PriorityQueue<>();
    
    for (int num : nums) {
        minHeap.offer(num);
        
        // 堆大小超过k，弹出最小的
        if (minHeap.size() > k) {
            minHeap.poll();
        }
    }
    
    // 堆顶就是第k大的
    return minHeap.peek();
}
```

## 边界 Case

**Case 1: 数组只有一个元素**
```
nums = [1], k = 1
堆: [1]，返回1 ✓
```

**Case 2: k等于数组长度**
```
nums = [3,2,1], k = 3
堆最后保留3个元素，堆顶是1（第3大）✓
```

**Case 3: 有重复元素**
```
nums = [3,3,3,3], k = 2
堆: [3,3]，返回3 ✓
```

**Case 4: 有负数**
```
nums = [-1,-2,-3], k = 2
排序: [-1,-2,-3]，第2大是-2
堆过程: [-1] → [-2,-1] → [-3,-1,-2]弹出-3 → [-2,-1]
返回-2 ✓
```
