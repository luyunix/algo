# DUI002 - 前K个高频元素 题解

## 思路是怎么想出来的

### 第一步：明确问题

```
问题：找出现频率前K高的元素
示例：nums = [1,1,1,2,2,3], k = 2
频率：1出现3次，2出现2次，3出现1次
前2高频：1, 2

输出：[1, 2]
```

### 第二步：分解问题

```
问题可以分解为两步：
1. 统计每个元素的频率
2. 找频率前K高的元素

第一步：哈希表统计
- key: 元素值
- value: 出现次数

第二步：用堆找前K高
- 用小顶堆，按频率排序
- 堆大小保持K
- 堆顶是第K高频的元素
```

### 第三步：为什么用小顶堆

```
和"第K大元素"类似：
- 我们要找前K高，意味着要淘汰频率低的
- 小顶堆堆顶是K个中最小的频率
- 遇到更高频的，替换堆顶
- 最后堆里就是前K高频
```

---

## 解法：哈希表 + 小顶堆

```java
public int[] topKFrequent(int[] nums, int k) {
    // 第一步：统计频率
    Map<Integer, Integer> freq = new HashMap<>();
    for (int num : nums) {
        freq.put(num, freq.getOrDefault(num, 0) + 1);
    }
    
    // 第二步：小顶堆（按频率排序）
    PriorityQueue<int[]> minHeap = new PriorityQueue<>(
        (a, b) -> a[1] - b[1]  // 按频率升序
    );
    
    for (Map.Entry<Integer, Integer> entry : freq.entrySet()) {
        minHeap.offer(new int[]{entry.getKey(), entry.getValue()});
        
        if (minHeap.size() > k) {
            minHeap.poll();  // 弹出频率最小的
        }
    }
    
    // 提取结果
    int[] result = new int[k];
    for (int i = 0; i < k; i++) {
        result[i] = minHeap.poll()[0];
    }
    
    return result;
}
```

## 边界 Case

**Case 1: 只有一个元素**
```
nums = [1], k = 1
频率: {1:1}
堆: [[1,1]]
返回 [1] ✓
```

**Case 2: 所有元素频率相同**
```
nums = [1,2,3,4], k = 2
频率: {1:1, 2:1, 3:1, 4:1}
堆保留任意2个即可，如[1,2] ✓
```

**Case 3: k等于不同元素个数**
```
nums = [1,1,2,2], k = 2
频率: {1:2, 2:2}
堆: [[1,2], [2,2]]
返回 [1,2] ✓
```
